# Android 定时消息真机审查

日期：2026-07-02
设备：ce8c42ac
范围：聊天页定时消息创建、展示、编辑、删除、键盘态布局

## 步骤与证据

1. 当前聊天页空定时状态
   - 截图：`01-current-chat-baseline.png`
   - 结果：没有定时任务时，输入框上方没有占位 loading。

2. 打开工具面板并进入定时消息
   - 截图：`02-after-plus-tap.png`、`03-time-picker.png`
   - 结果：定时入口可达，时间选择器默认时间可用。

3. 选择发送时间并输入内容
   - 截图：`04-send-time-selected.png`、`05-text-entered-before-send.png`
   - 结果：发送前只显示底部“发送时间”输入态，尚未创建服务端任务。

4. 创建定时任务
   - 截图：`06-after-scheduled-send.png`
   - 结果：创建后底部“发送时间”收起，输入框上方只保留一条单行任务提示，不显示消息内容。

5. 编辑定时任务
   - 截图：`07-edit-mode.png`
   - 结果：点击编辑后内容回填到输入框，显示“正在编辑定时消息”输入态。

6. 过期任务删除异常
   - 截图：`10-delete-1s.png`、`11-delete-10s.png`
   - 结果：服务端返回 `SCHEDULED_MESSAGE_NOT_CANCELABLE` 后，旧实现会恢复 stale 任务。已修复为不恢复 stale 任务并刷新列表。

7. 重新安装后验证 stale 任务消失
   - 截图：`12-after-reinstall-no-stale.png`
   - 结果：过期任务已不再显示，实际消息已按 00:30 投递到消息流。

8. 正常未来任务创建与删除
   - 截图：`15-picker-final.png`、`16-created-future-task.png`、`17-delete-future-1s.png`、`18-delete-future-6s.png`
   - 结果：01:00 未来任务创建成功；点击 X 后 1 秒内只显示“已删除定时消息”toast，6 秒后定时提示完全消失。

## UX 结论

- 当前核心逻辑闭环正确：无任务不占位、创建后单行提示、编辑可回填、删除可移除、到点任务不再恢复 stale UI。
- 单行提示的视觉方向合理，信息量克制，符合移动 IM 的输入区优先原则。
- 仍有一个可优化点：编辑态底部条“正在编辑定时消息：7月2日（周四）01:00”在键盘态占用较高，文字偏长。建议后续压缩为更短的单行模式，例如“编辑定时 01:00”，并把完整日期放到时间选择器或轻提示里。
- 选择时间底部弹层的关闭按钮在可访问性树里没有语义文本，后续可以补 `tooltip/semanticLabel`。

## 验证限制

- 本次覆盖 Android 真机单账号单会话。
- 未覆盖多端同步、iOS、弱网中断、服务端到点失败事件。
