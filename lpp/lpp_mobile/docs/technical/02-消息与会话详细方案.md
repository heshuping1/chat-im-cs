# 消息与会话详细方案

状态：当前有效

日期：2026-06-07

适用范围：`lpp_mobile`

## 1. 目标

APP 消息系统同时支持普通 IM 和在线客服，但两者入口、权限和状态必须清晰分离。

## 2. 会话类型

| 类型 | APP 处理 |
| --- | --- |
| `direct` | 普通单聊，进入消息首页。 |
| `group` | 普通群聊，进入消息首页。 |
| `temp_session` | 在线客服访客临时会话，只进入客服工作台。 |
| `direct_customer` / `im_direct` | 注册客户与客服直连 IM，按普通消息或客户会话入口处理。 |

客服会话和普通 IM 在业务域上保持隔离；发送生命周期、媒体上传、视频封面、待发恢复、本地消息持久化等端侧消息能力必须复用同一套实现，不能在客服页面单独复制媒体上传逻辑。

## 3. 核心不变量

- `temp_session` 不进入普通消息首页。
- 客服未接入、AI 未接管、历史终态、管理只读时不能发送。
- 自己发送的消息不产生未读。
- Gateway 重连后做增量同步，不全量重拉历史。
- 发送必须使用 `clientMsgId` 乐观插入。
- 服务端拒绝不能本地显示成功。
- 客户画像缺字段不得展示假数据。

## 4. 发送闭环

```text
输入 -> 生成 clientMsgId -> 本地乐观插入 sending -> 调发送接口 -> 成功替换服务端字段 -> 失败保留 failed/可重试
```

媒体发送入口：

- 相册入口必须支持图片和视频选择，选择结果先进入输入栏 presentation model，再由页面分派为 `MessageType.image` 或 `MessageType.video`。
- 拍摄入口必须直接进入单一拍摄页，遵循微信式交互：轻点快门拍照，长按快门录视频，录屏不在聊天拍摄入口范围内。
- 发送侧本地图片发送后必须第一时间插入消息流，先展示可识别的本地预览，并在图片上以轻量覆盖层体现上传/发送进度；上传成功替换远端资源时，端侧必须保留 `localPreviewUrl`，避免远端图加载期间回退灰色占位。
- 发送侧本地视频发送后必须第一时间插入消息流；发送前应生成端侧封面图，展示层优先使用本地封面，再回退服务端 `thumbnailUrl`，最后才显示视频占位，并在视频气泡上以覆盖层体现上传/发送进度。
- 图片和视频气泡参考微信式尺寸收敛：图片按资源真实宽高比展示，在移动端最大宽高内约束，不固定裁成正方形；视频默认按 16:9 紧凑比例展示。视频缺少 `localPosterUrl` / `thumbnailUrl` 时，端侧优先基于本地或已缓存视频生成首帧封面，失败后只显示浅色视频占位，不能出现黑屏式大色块。
- 文件上传/发送进度参考 PC 端内嵌方案：文件在文件卡右侧图标内展示圆弧进度；不使用消息外侧发送圆圈、不整体置灰、不恢复底部进度条。本轮只做进度展示，不提供暂停、继续、取消控制。
- 发送交互参考微信：图片、视频、文件、语音和文本不使用聊天页全局发送锁；一条媒体正在上传时，用户仍可继续发送下一条媒体或文本。每条消息独立维护上传/发送状态；位置选择、失败重发等状态敏感单动作可以保留重复点击保护。
- 发送侧上传中的图片、视频和文件不允许等传输完成后才显示；确认成功后替换服务端字段，视频继续保留 `localPosterUrl` 作为封面兜底。
- 本地消息库必须使用端侧持久化序列化保存 `localPreviewUrl`、`localPosterUrl` 等本地展示字段；接口请求仍使用 API 序列化，不能把本机路径传给服务端。
- 发送失败且属于可恢复网络/上传失败时，待发队列必须保存原始本地媒体 body；网络恢复重发前先上传本地媒体，视频还要上传本地封面并写入远端 `thumbnailUrl`。
- 待发队列恢复成功后必须回写本地消息库并按 `clientMsgId` 替换旧乐观/失败消息，避免重进聊天出现失败消息和成功消息重复。
- 手动重发失败消息必须沿用原 `clientMsgId`，不能重新生成发送 ID。
- 服务端相对路径如 `/media`、`/api`、`/uploads`、`/files` 必须按远端资源处理，不得误判为本地文件路径导致预览卡住或重复上传。
- 接收侧不展示发送方本地上传半成品和上传进度；只有服务端/Gateway 已下发完整消息后才进入消息流，端侧再按已收到的远端资源展示缩略、占位、下载或打开状态。
- 聊天本地消息库、会话快照、发送队列、媒体本体、本地搜索索引、存储治理和启动恢复按 `technical/06-APP微信级本地数据方案.md` 执行；其中接收侧图片、视频、文件、语音列表只展示预览，查看/播放/打开时通过 SQLite 媒体账本、持久文件目录和 Dio 下载器按需下载，已下载媒体 App 重启后必须可复用。
- 上传、文件读取、权限和平台能力归 runtime/service 层或既有平台能力封装处理，UI 不直接解释服务端 DTO。

## 5. Gateway 闭环

```text
connected -> disconnected/reconnecting -> 网络恢复 -> reconnect -> 增量 sync -> 合并会话/消息/已读
```

## 6. 聊天资料页检索

- 单聊和群聊资料页的“查找聊天内容”必须进入会话内搜索，不得退回全局搜索。
- 会话内搜索顶部参考微信：左侧不显示返回箭头，搜索输入框右侧显示“取消”，点击取消返回资料页。
- 会话内搜索空态展示快捷查找：日期、图片及视频、文件、链接；这些快捷入口只在当前 `conversationId` 范围内生效。
- 消息搜索接口使用 `/api/client/v1/search/messages`；会话内搜索必须传 `conversationId`，关键字为空时允许按快捷类型进入结果页，端侧再按消息类型或日期做轻量过滤。
- 会话内搜索结果必须展示真实内容：文本/链接展示正文摘要，文件展示真实文件名和大小，图片展示缩略图，视频展示封面、播放图标和时长；不得用 `[消息]`、`[文件]`、`[视频]`、`[图片]` 作为结果内容。
- 图片及视频快捷入口使用媒体网格展示，图片优先 `thumbnailUrl` 再 `url`，视频优先 `thumbnailUrl`；服务端未返回可展示资源时只显示轻量占位，不伪造历史封面。
- 搜索结果按用户时区做日期分组，标题显示 `今天`、`昨天`、`本周` 或 `yyyy年M月d日`，单条结果右侧保留具体时间。
- 全局搜索继续保留用户/消息 Tab，不展示会话内快捷查找。

## 7. 在线客服状态

```text
排队 -> 接入 -> 人工接待中 -> 关闭 -> 历史只读
AI 接待 -> 人工接管 -> 人工接待中
终态 -> 只读，禁止发送
```

## 8. 验收重点

| 场景 | 期望 |
| --- | --- |
| 普通会话列表 | 只展示 `direct/group`，不展示 `temp_session`。 |
| 文本发送 | 乐观插入，成功替换，失败可见可重试。 |
| 媒体发送 | 立即插入本地可见气泡，本地预览和待发 body 入库，重进会话不出现空加载。 |
| 媒体接收 | Gateway/接口完整消息落库后再显示，端侧按远端缩略图/文件信息渲染。 |
| 已读未读 | 自发不提醒，进入会话清未读。 |
| Gateway 重连 | 恢复后增量同步，不重复不丢关键消息。 |
| 客服排队 | 未接入前只能接入，不能发消息。 |
| 管理只读 | 管理员/所有者不能第三人发言。 |
| 推送点击 | 恢复正确空间并跳到目标会话。 |
| 聊天资料页查找 | 输入框右侧为取消，快捷查找可见，请求携带当前 `conversationId`，结果显示真实内容和日期分组。 |
