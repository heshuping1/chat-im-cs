# 设置与诊断详细方案

状态：当前有效

日期：2026-06-06

适用范围：`lpp_mobile`

## 1. 目标

设置和诊断为 APP 提供用户可控的通知、隐私、安全、语言、显示、聊天偏好和问题排查能力。它们必须服务真实运行和交付验收，不做只在 UI 上可见的假配置。

## 2. 能力边界

| 能力 | APP 要求 |
| --- | --- |
| 通知设置 | 保存到真实接口或明确展示同步失败；免打扰与推送角标口径一致。 |
| 隐私安全 | 黑名单、设备、账号安全等操作必须有权限和错误态。 |
| 语言显示 | 多语言、字号、显示设置需要兼容小屏和长文本。 |
| 聊天设置 | 自动翻译、媒体、清理缓存等必须说明本地/服务端副作用。 |
| 诊断 | breadcrumbs、sessionId、clientTraceId、requestId、导出诊断包。 |
| 反馈 | 用户反馈可携带诊断信息，敏感字段默认脱敏。 |
| 版本更新 | 启动和关于页调用匿名版本检查接口；强制更新阻断继续使用，可选更新允许稍后；Android 下载安装包必须校验 SHA-256，iOS 只打开外部更新链接。 |

## 3. Runtime owner

| Runtime 能力 | Owner 要求 |
| --- | --- |
| 通知 | 统一通知 service/adapter，不在页面散落平台调用。 |
| 存储 | Secure Storage、Hive、SQLite 按用途分离。 |
| 文件 | 导出、保存、分享走统一文件能力。 |
| 权限 | 相机、麦克风、相册、通知权限走统一权限服务。 |
| 版本更新 | `features/app_update` 负责接口防腐、版本策略、启动遮罩、关于页检查和平台安装/跳转能力。 |
| 诊断 | 日志、trace、错误上报和导出走诊断 owner。 |

## 4. 验收重点

| 场景 | 验收 |
| --- | --- |
| 保存设置 | 成功、失败、弱网、重试、回滚或重新拉取。 |
| 通知点击 | 前台、后台、锁屏、杀进程进入正确空间和页面。 |
| 诊断导出 | 文件可生成、可分享、敏感字段已脱敏。 |
| 错误上报 | `X-Client-Trace-Id`、requestId、sessionId 可串联排查。 |
| 权限拒绝 | 拒绝、永久拒绝、设置页引导和恢复后继续流程。 |
| 版本更新 | 无更新、接口失败放行、可选更新稍后、强制更新不可关闭、Android hash 错误、iOS 外链打开失败。 |
