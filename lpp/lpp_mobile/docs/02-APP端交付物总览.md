# APP 端交付物总览

状态：当前有效

日期：2026-06-06

适用范围：`lpp_mobile`

## 1. 文档定位

本文件回答“当前 APP 能交付什么、不能交付什么、交付还差什么”。详细状态以 `03-APP端功能矩阵.md`、`requirements/02-APP端未实现需求清单.md` 和 `release/` 为准。

## 2. 当前可作为交付基础的能力

| 能力 | 当前结论 |
| --- | --- |
| Flutter 工程 | 已具备 Android 优先运行、自动化测试和 debug 构建基础。 |
| 登录与空间 | 代码已验证，仍需真实账号、多企业和长期登录态专项。 |
| 普通 IM | 消息、会话、缓存、Gateway、发送生命周期已有代码和自动化基础。 |
| 通讯录与群 | 好友、组织、群资料和基础治理已有实现，仍缺多账号真机闭环。 |
| 在线客服 | 工作台、线程、接入、接管、关闭和只读状态已有实现基础，仍缺真实端到端。 |
| 管理入口 | 所有者工作台旧包真机烟测可进入，仍缺真实接口和权限专项。 |
| 设置诊断 | 设置、通知、诊断导出和错误上报已有实现基础。 |

## 3. 当前不可直接承诺的交付

- 新构建 APK 未完成真机覆盖安装验证。
- Android P0 主链路未用稳定真实账号完整跑完。
- 多角色入口和权限边界未逐角色真机关闭。
- admin token、tenant token 和 admin 401 重签发缺 APP 内真实接口日志专项。
- 在线客服 `temp_session` 端到端、并发接管、历史只读和多端同步未关闭。
- 推送、通话后台、锁屏、小窗、音频路由和大数据性能仍缺专项证据。

## 4. 发布判断

当前结论：不建议直接交付。

发布前必须进入 `06-APP端发布文档.md`，按 `release/01-发布检查清单.md` 和 `release/02-发布回归范围.md` 关闭 S0/S1 风险。可延期项必须由产品明确确认，并保留在 `requirements/02-APP端未实现需求清单.md`。
