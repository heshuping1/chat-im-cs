# APP 端发布文档

状态：当前有效

日期：2026-07-01

适用范围：`lpp_mobile`

## 1. 文档定位

本文件是 APP 发布总入口。发布前检查、回归范围和版本记录分别进入 `release/`。

## 2. 快速读取

| 需要回答 | 读取文档 |
| --- | --- |
| 当前能不能发版 | `release/01-发布检查清单.md` |
| 发布前要回归哪些范围 | `release/02-发布回归范围.md` |
| 本版本变更和验证证据在哪里 | `release/03-版本变更记录.md` |
| 正式版 Android/iOS 必跑哪些用例 | `release/04-正式版测试用例.md` |
| 正式版测试结果报告怎么写 | `release/05-正式版测试报告模板.md` |
| 发布问题如何重复跟踪 | `release/06-发布问题跟踪台账.md` |
| 未关闭风险是什么 | `requirements/02-APP端未实现需求清单.md` |
| 功能状态是否更新 | `03-APP端功能矩阵.md` |

## 3. 当前发布判断

当前结论：不建议直接交付。

原因：自动化和部分 Android 单机真机验证已有证据，但新构建包真机安装、Android P0 主链路、多角色真实账号、在线客服端到端、admin token 真接口、推送后台、通话专项和大数据性能仍未完成。

## 4. 发布前硬门槛

1. S0 风险全部关闭。
2. P0 主链路有 Android 真机证据。
3. 角色、token、空间和权限边界完成专项。
4. 在线客服端到端完成真实联调。
5. 推送或通话如果纳入本次交付，必须有真机专项证据。
6. Android 和 iOS 都必须按 `release/04-正式版测试用例.md` 完成正式版回归。
7. 每次正式版必须按 `release/05-正式版测试报告模板.md` 产出测试结果报告。
8. 失败、阻塞、重复出现或延期接受的问题必须登记到测试报告，并同步到 `release/06-发布问题跟踪台账.md`。
9. `release/03-版本变更记录.md` 记录版本、范围、验证、风险、测试报告路径和回滚口径。

## 5. 正式版测试交付物

每次正式版发布前必须产出以下交付物：

| 交付物 | 要求 |
| --- | --- |
| Android 测试结果 | 至少覆盖 P0 主链路、安装启动、权限、弱网和本次变更相关专项。 |
| iOS 测试结果 | 至少覆盖 P0 主链路、安装启动、权限、弱网和本次变更相关专项。 |
| 正式版测试报告 | 使用 `release/05-正式版测试报告模板.md`，记录通过、失败、阻塞、不适用和证据。 |
| 发布问题台账 | 所有失败、阻塞、重复出现、延期接受问题同步到 `release/06-发布问题跟踪台账.md`。 |
| 发布判断 | 明确 `可发布` / `不可发布` / `带风险发布` / `仅限灰度`。存在未关闭 S0/S1 时默认不可发布。 |
