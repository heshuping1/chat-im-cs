# App 技术架构方案

更新时间：2026-06-06

完整技术方案见 `../../../docs/03-技术方案.md`。

## 1. 技术目标

- 移动 IM 主链路稳定。
- 在线客服移动接待状态闭环。
- token、空间、权限和缓存严格隔离。
- Android 真机、弱网、推送、通话和媒体能力可验证。
- 后续 AI 和工程师能按 owner 稳定接手。

## 2. 技术栈

| 层级 | 技术 |
| --- | --- |
| 跨端框架 | Flutter / Dart |
| 状态管理 | Riverpod 2 |
| 路由 | GoRouter |
| HTTP | Dio + `core/network/http_client.dart` |
| Admin HTTP | `core/admin/admin_http_client.dart` |
| 实时消息 | SignalR / WebSocket，路径 `/ws/client` |
| 音视频 | `flutter_webrtc`，路径 `/hubs/voicecall` |
| 本地缓存 | Hive |
| 本地数据库 | SQLite / sqflite |
| 安全存储 | Flutter Secure Storage |
| 通知 | flutter_local_notifications + 远程推送 adapter 待专项 |

## 4. Token 和空间

| Token | 用途 |
| --- | --- |
| platformToken | 登录、租户、空间选择、签发 admin token。 |
| tenant accessToken | `/api/client/v1/*`、`/ws/client`、`/hubs/voicecall`。 |
| refreshToken | 刷新当前空间 accessToken。 |
| admin accessToken | `/api/admin/v1/*`。 |

要求：

- 会话、消息、草稿、未读、token 必须按 `spaceId` 隔离。
- 切换空间后不能复用上个空间缓存、Gateway 连接或 admin token。
- 普通 tenant token 不调用 admin API。
- admin 401 后清理 admin token，并用 platform token 重新签发。

## 5. 技术风险

| 风险 | 说明 |
| --- | --- |
| token 混用 | admin API、client API、platform API 必须分域。 |
| 空间缓存串线 | 切换空间后会话、未读、Gateway、token 不得串用。 |
| `temp_session` 污染消息首页 | 临时客服会话只能在客服工作台展示。 |
| 管理只读误发言 | 管理员/所有者第三人查看必须禁用输入区和写接口。 |
| 媒体和通话真机差异 | 图片、文件、语音、视频、通话、权限、后台必须真机专项。 |
| 大列表性能 | 会话、消息、联系人、客户、客服线程必须关注懒加载、分页和缓存。 |
