# App UI 与交互方案

更新时间：2026-06-06

完整跨端 UI 总账见 `../../../docs/07-UI与交互设计说明.md`。

## 1. 设计目标

LPP App 要以微信级移动 IM 体验为基准：清晰、克制、可快速操作、弱网可恢复、输入自然、列表流畅、状态闭环完整。

移动端不照搬 PC 三栏工作台，不把管理报表和大表格硬塞进手机。App 的核心优势是即时沟通、移动接待、快速查看、扫码、通话、通知和轻量处理。

## 2. 全局原则

- 主导航只放高频入口。
- 低频能力进入“我”、设置、工作台、会话详情或具体业务页。
- 客户不暴露内部管理语义。
- 客服工作台服务移动接待，不做 PC 指挥台。
- 状态必须分离：IM 在线状态、客服接待状态、单条会话状态、客户在线状态不能混用。
- 所有操作补齐 loading、empty、error、disabled、permission、offline。
- 文案贴近用户任务，不暴露内部实现名。
- 长文本、长昵称、长群名、低端机和小屏必须可读不遮挡。

## 3. 页面体系

| 页面域 | App 形态 |
| --- | --- |
| 消息首页 | 微信式消息列表，顶部标题、空间/账号入口、圆形 `+`、独立搜索框、白底平铺会话行。 |
| 聊天页 | 消息虚拟化、本地缓存优先、输入区自然、媒体缓存、发送失败可重试。 |
| 通讯录 | 好友、组织、群、客户按角色展示，避免管理语义污染普通角色。 |
| 客服工作台 | 在线客服、客户服务、效能；当前接待/历史只读；排队、进行中、VIP、搜索。 |
| 客户资料 | 底部弹层或摘要页，展示基础信息和关键状态，不放完整交易大表。 |
| 管理工作台 | 管理员/所有者轻量管理入口，复杂治理进入后续 PC/管理扩展。 |
| 设置/我的 | 个人资料、二维码、隐私、安全、通知、字号、语言、诊断、退出登录。 |
| 通话 | 语音/视频状态清晰，权限拒绝可恢复，小窗不遮挡核心按钮。 |

## 4. 消息和输入体验

- 进入聊天页后本地立即清未读，服务端已读只在必要时上报。
- WebSocket 重连后做增量同步，不全量重拉历史。
- 消息发送使用乐观插入和 `clientMsgId`。
- 输入栏右侧固定保留表情和 `+`；文本发送由键盘右下角完成。
- 禁言、只读、未接入、AI 未接管、历史终态时，输入区必须禁用并解释原因。
- 媒体消息要考虑鉴权加载、缓存、失败重试和权限拒绝。
- 管理员/所有者只读查看时，不允许出现可发送的错觉。

## 5. 在线客服交互

| 状态 | App 表现 |
| --- | --- |
| 排队 | 展示 `接入`，输入区禁用。 |
| AI 接待 | 展示 `人工接管`，接管前输入区禁用。 |
| 我正在接待 | 展示 `继续沟通`，可发送。 |
| 其他客服接待 | 普通客服只读或展示正在接待人。 |
| 已结束/历史 | 展示 `查看`，只读，不能接入、接管、关闭或发送。 |

进入在线客服页只读取 `reception/status`，不能自动上线。只有客服主动切换在线/忙碌/短暂离开/离线时才更新服务端状态。
