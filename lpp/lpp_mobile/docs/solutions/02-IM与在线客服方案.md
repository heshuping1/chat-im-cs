# App IM 与在线客服方案

更新时间：2026-06-06

## 1. 目标

App 端消息系统同时支持普通 IM 和在线客服，但两者入口、权限和状态必须清晰分离。

核心目标：

- 普通 `direct/group` 会话进入消息首页。
- 在线客服 `temp_session` 只进入客服工作台。
- 客服临时会话可复用聊天承载，但必须携带线程上下文和禁发状态。
- Gateway、API、缓存和本地状态都按 `spaceId` 隔离。
- 弱网下优先本地可读，恢复后增量同步。

## 2. 核心不变量

- `temp_session` 不进入普通消息首页。
- 客服未接入、AI 未接管、历史终态、管理只读时不能发送。
- 自己发送的消息不产生未读。
- Gateway 重连后做增量同步，不全量重拉历史。
- 发送必须使用 `clientMsgId` 乐观插入。
- 服务端拒绝不能本地显示成功。
- 客户画像缺字段不得展示假数据。

## 3. 会话类型

| 类型 | App 处理 |
| --- | --- |
| `direct` | 普通单聊，进入消息首页。 |
| `group` | 普通群聊，进入消息首页。 |
| `temp_session` | 在线客服访客临时会话，只进入客服工作台。 |
| `direct_customer` / `im_direct` | 注册客户与客服直连 IM，按普通消息/客户会话入口处理。 |

## 4. 消息发送闭环

```text
输入 -> 生成 clientMsgId -> 本地乐观插入 sending -> 调发送接口 -> 成功替换服务端字段 -> 失败保留 failed/可重试
```

## 5. Gateway 重连闭环

```text
connected -> disconnected/reconnecting -> 网络恢复 -> reconnect -> 增量 sync -> 合并会话/消息/已读
```

## 6. 在线客服状态

```text
排队 -> 接入 -> 人工接待中 -> 关闭 -> 历史只读
AI 接待 -> 人工接管 -> 人工接待中
终态 -> 只读，禁止发送
```
