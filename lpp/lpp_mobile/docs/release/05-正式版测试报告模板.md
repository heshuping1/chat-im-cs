# APP 正式版测试报告模板

状态：当前有效

日期：2026-07-01

适用范围：`lpp_mobile` Android / iOS 正式版测试报告

## 1. 使用规则

1. 每次正式版、候选正式版或灰度正式版发布前，必须复制本模板生成当次测试报告。
2. 报告建议存放到 `reports/mobile/release/YYYYMMDD_<version>/summary.md`，并在 `release/03-版本变更记录.md` 追加证据链接。
3. Android 和 iOS 必须分别记录设备、系统、构建、账号、结果和证据。
4. 失败、阻塞、反复出现或延期接受的问题必须写入本文档的“问题清单”，并同步到 `06-发布问题跟踪台账.md`。
5. 报告内所有 token、手机号、邮箱、验证码、cookie、Authorization 必须脱敏。

## 2. 报告头

```text
报告编号:
版本/构建:

```
