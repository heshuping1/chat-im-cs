# APP 发布回归范围

状态：当前有效

日期：2026-07-01

适用范围：`lpp_mobile`

## 1. P0 回归

正式版 P0 回归必须同时覆盖 Android 和 iOS。详细用例编号、步骤、预期和证据要求见 `04-正式版测试用例.md`。

| 路径 | 必测内容 |
| --- | --- |
| 登录和空间 | 邮箱/微界号登录、租户选择、个人空间、企业空间、退出登录、token 刷新。 |
| 消息 | 会话列表、文本、图片、文件、语音、失败重试、未读、已读、搜索、收藏、翻译。 |
| 好友 | 搜索、扫码、申请、同意、拒绝、通过后会话出现。 |
| 通讯录 | 组织、成员、客户、群、角色可见范围。 |
| 群 | 创建群、添加成员、群资料、禁言、申请、公告、退出/解散。 |
| 在线客服 | 排队、接入、人工接管、继续沟通、关闭、历史只读、来源渠道、客户摘要。 |
| 管理 | 管理员/所有者只读查看、客户管理、客服中心、群监管入口。 |
| 设置诊断 | 通知、隐私、安全、语言、诊断导出、退出登录。 |

## 2. 设备专项

正式版设备专项必须分别记录 Android 和 iOS 测试结果。任一平台未测必须在正式版测试报告中登记为 `未验证` 或 `不适用`，并说明原因。

| 专项 | 必测内容 |
| --- | --- |
| Android 真机 | 安装、启动、登录、前后台切换、权限弹窗、截图证据。 |
| iOS 真机 | 安装、启动、登录、前后台切换、权限弹窗、截图证据。 |
| 弱网 | 断网提示、恢复、Gateway 重连、离线补偿、发送重试。 |
| 推送 | 前台、后台、锁屏、杀进程、点击进入正确空间和页面。 |
| 通话 | 呼叫、接听、拒绝、挂断、后台、锁屏、音频路由。 |
| 媒体 | 相机、相册、文件、录音、上传失败、鉴权失效。 |
| 性能 | 500+ 会话、长聊天、低端机、长时间后台恢复。 |

## 3. 正式版报告要求

每次正式版发布前必须使用 `05-正式版测试报告模板.md` 生成当次测试报告。报告必须包含：

1. Android 和 iOS 的设备、系统、构建、账号、角色和空间。
2. 自动化命令、构建命令、真机操作和结果。
3. P0/P1 用例通过、失败、阻塞、不适用汇总。
4. 失败和阻塞项的问题编号、复现步骤、实际结果、预期结果、证据和负责人。
5. 发布结论：`可发布` / `不可发布` / `带风险发布` / `仅限灰度`。

## 4. 问题登记要求

1. 失败或阻塞项必须在测试报告的问题清单中登记。
2. 测试报告中的问题必须同步到 `06-发布问题跟踪台账.md`。
3. 重复问题沿用原问题编号，并在台账复测历史中追加记录。
4. S0/S1 未关闭时默认不可发布；若带风险发布，必须写明风险接受人、影响范围、回滚方案和目标修复版本。
