# APP 发布检查清单

状态：当前有效

日期：2026-06-06

适用范围：`lpp_mobile`

## 1. 当前发布判断

当前结论：不建议直接交付。

原因：新构建包真机安装、Android P0 主链路、多角色真实账号、在线客服端到端、admin token 真接口、推送后台、通话专项和大数据性能仍未完成。

## 2. 发布前硬检查

| 检查项 | 当前状态 | 发布要求 |
| --- | --- | --- |
| Android P0 主链路 | 未关闭 | 登录、空间、消息、通讯录、客服、设置真机通过。 |
| 多角色入口 | 未关闭 | 客户、员工、客服、管理员、所有者入口和权限符合边界。 |
| Token 域 | 未关闭 | admin/client/platform token 不混用，401 重签发可验证。 |
| 管理只读 | 未关闭 | 第三人查看输入禁用，写接口服务端拒绝。 |
| 在线客服端到端 | 未关闭 | 访客发起、客服接入、接管、关闭、历史只读、多端同步。 |
| 推送通话 | 未关闭 | 纳入交付范围时必须完成真机专项。 |
| 性能稳定性 | 未关闭 | 大列表、低端机、长稳有基线。 |
| 版本记录 | 待更新 | `03-版本变更记录.md` 补齐范围、验证、风险和回滚。 |

## 3. 发布前不能出现

- 任一主要角色无法登录。
- 普通消息无法发送。
- 客户能进入管理员/所有者工作台。
- admin API 使用普通 tenant token。
- 管理员/所有者能以第三人视角向客户会话发消息。
- 在线客服 `temp_session` 混入普通消息首页。
- 群禁言权限与产品规则相反。
- 客服无法处理客户咨询。
- P0 路径频繁崩溃、白屏或卡死。
- 纳入交付范围的通话/推送能力没有真机验收。
