# App 测试验收方案

更新时间：2026-06-06

完整验收总账见 `../../../docs/04-测试验收方案.md`。

## 1. 验收目标

App 验收不只看“能运行”，还要证明移动端主链路在真实账号、真实空间、真实服务端、真实 Android 设备、弱网和多角色下可用。

## 2. 默认轻量验证

| 改动类型 | 默认验证 |
| --- | --- |
| 纯文档或方案 | 文档链接、状态口径和工作区状态检查。 |
| 单组件 UI 或低风险逻辑 | `flutter analyze --no-fatal-infos` 或定向 `flutter test`。 |
| 单 Provider/UseCase/Datasource | 定向测试对应目录或文件。 |
| 入口、权限、token、消息、客服、缓存、推送、通话 | 升级到重量级验证或先说明需要升级。 |

## 4. Android 真机记录要求

每次真机验收必须记录：

- 设备型号和 Android 版本。
- App 构建类型和版本。
- 测试账号、角色、空间。
- 网络环境。
- 执行时间。
- 通过项、失败项、日志、requestId。
- 截图或录屏路径。

## 5. P0 必测路径

| 路径 | 必测内容 |
| --- | --- |
| 登录和空间 | 邮箱/微界号登录、租户选择、个人空间、企业空间、退出登录、token 刷新。 |
| 消息 | 会话列表、文本、图片、文件、语音、失败重试、未读、已读、搜索、收藏、翻译。 |
| 好友 | 搜索、扫码、申请、同意、拒绝、通过后会话出现。 |
| 通讯录 | 组织、成员、客户、群、角色可见范围。 |
| 群 | 创建群、添加成员、群资料、禁言、申请、公告、退出/解散。 |
| 在线客服 | 排队、接入、人工接管、继续沟通、关闭、历史只读、来源渠道、客户摘要。 |
| 管理 | 管理员/所有者只读查看、客户管理、客服中心、群监管入口。 |
| 通话 | 呼叫、接听、拒绝、挂断、后台、锁屏、音频路由。 |
| 推送 | 前台、后台、杀进程、点击进入正确空间和会话。 |
| 弱网性能 | 断网提示、恢复、Gateway 重连、离线补偿、长列表、低端机内存。 |
