## 4. 发布前不能出现

- 任一主要角色无法登录。
- 普通消息无法发送。
- 客户能进入管理员/所有者工作台。
- admin API 使用普通 tenant token。
- 管理员/所有者能以第三人视角向客户会话发消息。
- 在线客服 `temp_session` 混入普通消息首页。
- 群禁言权限与产品规则相反。
- 客服无法处理客户咨询。
- P0 路径频繁崩溃、白屏或卡死。
- 纳入交付范围的通话/推送能力没有真机验收。

## 5. 下一轮建议顺序

1. 跑 Android P0 主链路：登录、空间、消息、通讯录、客服、设置。
2. 做 admin token 真接口专项：管理员/所有者账号、admin API、401 重签发、拒绝态。
3. 做在线客服端到端：访客发起、客服接入、人工接管、关闭、历史只读、多端同步。
4. 做好友和群多账号闭环：申请、审批、群禁言、群申请、退出/解散。
5. 做推送、通话和后台：前后台、锁屏、通知点击、音频路由。
6. 建大数据性能基线：500+ 会话、长聊天、低端机、长时间后台恢复。

## 6. 可延期项

- 深色模式完整打磨。
- 图片/视频压缩优化。
- 消息撤回后重新编辑。
- 面对面建群。
- 完整交易/资金/工单处理流。
- 质检审计、运营分析、批量管理。
