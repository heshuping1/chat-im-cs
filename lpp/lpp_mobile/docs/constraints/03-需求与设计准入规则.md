# APP 需求与设计准入规则

状态：当前有效

日期：2026-06-06

适用范围：`lpp_mobile`

## 1. 需求状态

| 状态 | 含义 | 是否允许开发 |
| --- | --- | --- |
| 草稿 | 只有想法或粗略方向。 | 否 |
| 评审中 | 产品、设计、接口或角色边界仍在讨论。 | 否 |
| 完成 | 范围、角色、状态、异常、验收和依赖已明确。 | 取决于设计状态 |
| 变更中 | 已完成需求正在被调整。 | 否，只允许明确修复或回滚 |
| 暂缓 | 当前版本不做，但保留记录。 | 否 |

## 2. 设计状态

| 状态 | 含义 | 是否允许开发 |
| --- | --- | --- |
| 草稿 | 只有技术方向或初步拆分。 | 否 |
| 评审中 | 模块边界、数据流、API 或验证方案未定。 | 否 |
| 完成 | 落点、分层、接口、状态、错误处理和验证方案已明确。 | 取决于需求状态 |
| 变更中 | 既有设计需要调整。 | 否，只允许明确修复或回滚 |
| 暂缓 | 当前版本不做或等待服务端。 | 否 |

## 3. 开发准入

只有同时满足以下条件才允许进入实现：

1. `需求状态 = 完成`。
2. `设计状态 = 完成`。
3. 需求已进入 `requirements/01-APP端需求池.md`。
4. 功能矩阵已新增或更新对应行。
5. 服务端 API 支持已确认，或阻塞项已进入 `05-服务端支持.md` 和未实现清单。
6. 验收方式已明确到自动化、接口联调、Android 真机、多账号、多端或发布回归范围。
