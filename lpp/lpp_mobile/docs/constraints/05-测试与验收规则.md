# APP 测试与验收规则

状态：当前有效

日期：2026-06-06

适用范围：`lpp_mobile`

## 1. 验收原则

APP 验收不只看能运行，还要证明真实账号、真实空间、真实服务端、Android 真机、弱网、多角色和多端关键路径可用。

## 2. 默认验证

| 改动类型 | 默认验证 |
| --- | --- |
| 纯文档或方案 | 文档链接、状态口径和工作区状态检查。 |
| 单组件 UI 或低风险逻辑 | `flutter analyze --no-fatal-infos` 或定向 `flutter test`。 |
| Provider / UseCase / Repository | 定向测试对应文件或目录。 |
| 入口、权限、token、消息、客服、缓存 | 升级到相关自动化和接口联调。 |
| 推送、通话、媒体、权限、后台 | Android 真机专项，必要时双真机。 |

## 3. 常用命令

```bash
cd /Users/treesoft/Downloads/lpp-flutte/lpp/lpp_mobile
flutter analyze --no-fatal-infos
flutter test ../scripts/mobile/test/flutter
../scripts/mobile/test/run_full_regression.sh
flutter run -d <deviceId>
flutter build apk --debug
```

## 4. 真机记录要求

每次真机验收必须记录：

- 设备型号和 Android 版本。
- App 构建类型和版本。
- 测试账号、角色、空间。
- 网络环境。
- 执行时间。
- 通过项、失败项、日志、requestId。
- 截图或录屏路径。
