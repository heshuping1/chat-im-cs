更新时间：2026-06-06

本文用于跟踪 App 当前还不能关闭的功能、专项验证和服务端依赖。完整跨端总账见 `../../../docs/11-项目现状与缺口清单.md`，服务端诉求总账见 `../../../docs/服务端支持.md`。

## 1. 跟踪口径

任务保持未完成，通常属于以下任一情况：

- 功能入口、状态或关键交互仍未实现。
- 代码已接入，但未用真实账号、真机、多端或真实服务端数据跑通。
- 接口、字段、事件、权限或错误码仍未由服务端稳定确认。
- 自动化或人工验收没有留下可复查证据。
- 发布风险仍需要产品或工程明确延期。

状态字段统一使用：

| 状态 | 含义 |
| --- | --- |
| 未开始 | 尚未进入实现、联调或验证。 |
| 进行中 | 已有实现或验证动作，但仍缺关闭条件。 |
| 待真机 | 主要差距是真机、多账号、多端或设备能力验证。 |
| 待联调 | 前端已接入接口，但真实数据、权限或字段没有闭环。 |
| 待服务端 | 主要阻塞来自服务端接口、字段、事件、配置或业务口径。 |
| 可延期 | 不影响当前 Android P0 交付，可由产品明确延期。 |
| 已完成 | 完成标准满足，并已补齐证据位置。 |

## 8. 更新规则

1. 每关闭一个任务，勾选完成列，把状态改为 `已完成`，并补充证据路径、执行账号角色、设备和日期。
2. P0/P1 状态变化后，同步更新 `02-功能矩阵与当前状态.md` 和 `../verification/02-发布风险与检查清单.md`。
3. 发现新缺口时，先判断是 App 客户端任务还是服务端依赖；不要把服务端未提供的能力写成 App 已完成。
4. 可延期任务必须由产品明确延期；延期后仍保留在本清单，避免后续遗忘。
