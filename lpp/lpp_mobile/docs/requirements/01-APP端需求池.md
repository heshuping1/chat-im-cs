# APP 端需求池

状态：当前有效

日期：2026-06-06

适用范围：`lpp_mobile`

## 1. 文档定位

本文件用于登记后续可继续开发、讨论或评审的 APP 需求。完整产品边界见 `01-产品需求规格.md`；功能状态见 `02-功能矩阵与当前状态.md`。

新增需求必须先进入本池，再按 `03-APP端需求状态流转规则.md` 和 `../constraints/03-需求与设计准入规则.md` 判断是否允许开发。

## 4. 维护规则

1. 需求进入开发前，必须把 `需求状态` 和 `设计状态` 补齐。
2. 如果沿用既有技术方案，也要把 `设计状态` 标为 `完成`，并注明沿用的 `technical/` 文档。
3. 如果服务端未支持，开发准入必须标为不允许或仅允许前置设计/防腐层准备。
4. 状态变化后同步 `03-APP端功能矩阵.md` 和 `02-APP端未实现需求清单.md`。
