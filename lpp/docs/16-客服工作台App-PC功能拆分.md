# 客服工作台 App/PC 功能拆分

更新时间：2026-05-24

本文用于承接客服工作台的产品规划和端侧拆分。当前 Flutter 仓库主要交付 App；PC 端路线已调整为：**Electron + React + TypeScript 的 Windows 本地客服客户端**。PC 客户端不部署线上 Web 站点，不做 macOS 客户端，不走 Flutter Desktop。PC 启动准备和复用边界见 `17-PC端准备与复用方案.md`。

## 1. 总体判断

客服接待不是单纯聊天，而是“聊天 + 客户上下文 + 快捷处理 + AI 辅助 + SLA”的工作台能力。App 和 PC 客服客户端应共享同一套服务端数据和权限口径，但展示密度和操作深度不同。

| 端侧 | 定位 | 核心目标 |
| --- | --- | --- |
| App | 移动轻量接待 | 快速看重点、快速回复、快速接管、移动补充处理。 |
| PC 客服客户端 | 客服主力工作台 | 多会话高频接待、客户全景、交易和工单联动、通知、托盘、文件和诊断能力。 |

## 2. 从参考图可借鉴的能力

| 参考能力 | App 借鉴方式 | PC 借鉴方式 |
| --- | --- | --- |
| 聊天区 + 客户上下文 | 聊天页顶部展示客户摘要，右上角进入客户画像底部弹层。 | 三栏或多栏常驻布局，右侧上下文面板不折叠。 |
| 客户画像 | 底部弹层摘要展示基础资料、标签、账户概况、历史交易、临时订单和工单摘要。 | 客户详情页完整展示总览、交易记录、资金流水、会话、工单、触达、KYC/合规、IB/设备。 |
| 快捷处理 | 输入区和 `+` 面板承载快捷话术、知识库、图片/附件、翻译、表情、定时发送。 | 输入区工具栏承载知识库、快捷话术、AI 起草、附件、图片、翻译、转人工、升级等高频动作。 |
| AI 辅助 | 展示 AI 自动接待状态、人工接管、AI 建议回复；人工转 AI 接待本期暂不做。 | 展示 AI 自动对话、AI 起草、情绪识别、意图识别、升级建议、质检线索；是否支持人工转 AI 由后续 PC 方案确认。 |
| SLA 和状态 | 聊天页轻量展示 SLA 倒计时、客户正在输入、发送失败和合规结果。 | 队列、会话列表和聊天区同步展示 SLA、等待时长、接待人、风险和质检状态。 |

## 3. App 端需要的功能

App 端目标是移动客服可完成一次轻量接待闭环，不承担完整 CRM、交易后台或运营分析。

| 功能 | App 口径 |
| --- | --- |
| 在线客服工作台 | 展示全部、排队、进行中、VIP 筛选；支持接入、人工接管、继续沟通、关闭会话。 |
| 聊天页轻量上下文 | 顶部展示客户姓名、VIP/风险/来源、AI 接待状态、SLA 倒计时。 |
| 客户画像弹层 | 展示基础资料、标签、账户概况、历史交易概况、临时订单数据、工单摘要；无数据展示空态，不造假。 |
| 输入区快捷能力 | 支持快捷话术、知识库、图片/附件、翻译、表情、定时发送。 |
| AI 辅助 | 支持 AI 自动接待状态、人工接管、AI 建议回复；人工转 AI 接待本期暂不做。 |
| 状态反馈 | 展示客户正在输入、发送失败、合规检查结果、接口不可用、无权限空态。 |
| 移动处理 | 适合临时接待、补充回复、查看客户重点信息和轻量关闭会话。 |

## 4. PC 端需要的功能

PC 是客服高频主战场，适合高密度信息和复杂操作。PC 能力不应被 App 当前实现范围限制。

| 功能 | PC 口径 |
| --- | --- |
| 三栏/多栏工作台 | 左侧会话/队列，中间聊天，右侧客户上下文面板常驻。 |
| 完整客户详情页 | 总览、交易记录、资金流水、会话、工单、触达、KYC/合规、IB/设备。 |
| 客户运营信息 | 标签画像、营销同意、客户生命周期、高净值客户、AI 情绪、最后活跃。 |
| 接待效率工具 | 知识库、快捷话术、AI 起草、转人工、分配客户经理、生成报告。 |
| 风控和交易上下文 | 历史交易概况、临时订单/持仓/挂单、资金、风险等级、KYC 状态。 |
| 团队管理 | 客服分配、接待监控、SLA、会话质检、审计追溯、批量筛选和报表。 |

## 5. 不建议放入 App 的功能

- 完整交易记录明细表、资金流水明细表。
- 大量工单处理流、报表生成、批量筛选。
- 完整客户运营分析、触达历史全量管理。
- 多客服绩效、质检、审计追溯。
- 复杂团队管理和批量运营动作。

App 可以提供入口、摘要或只读预览，完整能力放 PC。

## 6. 数据和接口口径

- App 和 PC 客服客户端必须使用同一套 `profile-card`、客户详情、客服线程、订单、工单、触达和权限数据。
- App 和 PC 必须复用同一套服务端接口合同、权限规则、状态口径、诊断字段和测试用例；PC 采用 React/TypeScript，不直接复用 Flutter widget/provider，但不能复制出另一套业务判断。
- `临时订单数据`：App 展示摘要列表；PC 展示完整订单表、筛选、详情和风险提示。
- 敏感字段按角色和端侧统一脱敏，不能出现 App 可见、PC 不可见或相反的权限错位。
- 服务端应提供 SLA、客户正在输入、合规检查、AI 建议、客户画像、客户详情 Tab 数量和临时订单字段的稳定口径。
- PC 端可以在同一页面并行拉取更多详情，但不能用另一套业务枚举或另一套权限判断。

## 6.1 复用实现要求

| 层级 | 是否复用 | 要求 |
| --- | --- | --- |
| domain/entity/usecase | 必须复用 | 会话、消息、客户、客服线程、权限、空间和画像模型不能复制一套。 |
| data/datasource/repository | 必须复用 | HTTP endpoint、请求体、响应解析和错误处理必须同源。 |
| provider/state | 规则复用 | App 继续使用 Riverpod；PC 端可使用自己的状态管理，但必须复用同一接口合同、状态机和权限测试夹具。 |
| presentation/page | 分端实现 | 移动端保留单栏和底部导航；PC 端新增 Electron Shell、三栏工作台、右侧上下文和桌面能力。 |
| platform adapter | 分端实现 | 扫码、推送、文件选择、通知、通话 UI、本地存储按平台适配。 |
| diagnostics | 必须复用 | App/PC 错误上报、脱敏、requestId 和 breadcrumbs 使用同一套机制。 |

当前已补公共 `ResponsiveLayout`：移动单栏、宽屏双栏、桌面三栏，后续 PC 客服工作台可直接复用。

## 7. 验收口径

| 验收项 | 端侧 | 标准 |
| --- | --- | --- |
| 在线客服轻量接待 | App | 客服进入在线客服会话后，可查看客户画像、临时订单、工单摘要，并能接管、回复、关闭。 |
| 空态 | App | 无临时订单时显示“暂无临时订单”，无工单时显示空态，不显示假数据。 |
| 上下文常驻 | PC | 聊天时右侧客户上下文面板常驻，客户详情各 Tab 数量和列表与数据一致。 |
| 权限脱敏 | App/PC | 客服、管理员、所有者看到的客户敏感字段不同，脱敏规则一致。 |
| 数据一致性 | App/PC | 同一客户的标签、风险、账户概况、订单、工单和接待状态在两端口径一致。 |
