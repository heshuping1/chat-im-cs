## 7. 交付判断

当前结论：**不建议直接交付**。

原因：

- 本轮已执行 `flutter test --concurrency=1 ../scripts/mobile/test/flutter`，202 个用例全部通过。
- 本轮已增强错误诊断：breadcrumbs、sessionId、clientTraceId、HTTP `X-Client-Trace-Id` 和本地诊断包导出已落地并通过测试。
- 本轮已隔离移动推送依赖，默认依赖不再包含 `firebase_core`、`firebase_messaging`、`jpush_flutter`；PC 端路线调整为 Electron Gateway 系统通知；macOS debug 构建仅作为历史体检记录。
- 本轮已执行 `flutter analyze --no-fatal-infos`，结果通过；默认 `flutter analyze` 仍有 179 条 info 级历史 lint。
- 本轮已构建、安装并启动 Android debug 包；历史 release 模式可安装包仍保留。
- 本轮已完成 API 脚本、当前登录态真机截图和在线客服临时会话模拟，但未完成 Android P0 主链路、多角色、客服账号在线客服端到端和后台专项。
- 高风险能力仍缺真实账号、多端、后台或服务端数据证据。
- 功能矩阵中服务端缺失项已单独整理到 `服务端支持.md`，App 侧剩余事项见 `APP-TODO-*`。

达到交付候选前，至少需要完成：

1. Android P0 真机回归
2. 在线客服 App 真机专项：临时会话、客户画像、临时订单、人工接管、关闭、消息首页不展示临时会话；人工转 AI 接待本期暂不验收
3. admin token 真接口专项
4. 客服/好友/群多账号专项
5. 通话和推送专项
6. 大数据、低端机和长稳性能基线记录
