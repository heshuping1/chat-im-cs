# LPP 性能稳定性基线

更新时间：2026-05-23

当前结论：已建立基础性能稳定性记录，但尚未形成完整交付基线。已记录 release 模式冷启动、热启动、基础内存和断网/恢复提示；本轮 debug 包诊断增强后重新安装启动，记录 `TOTAL PSS` 约 `382MB`；仍缺大数据长列表、低端机、后台长稳、推送、通话和弱网消息实时性专项。

## 1. 目标

性能稳定性基线用于判断 App 在真实 Android 设备、真实网络、真实数据规模下是否具备交付条件。

## 2. 必须建立的指标

| 指标 | 目标值 | 当前状态 | 验证方式 |
| --- | --- | --- | --- |
| 冷启动时间 | 低端机也应可感知流畅；正式目标待压测后锁定 | 已记录：release 模式最新冷启动 `TotalTime=343ms`，历史 `426ms/436ms/429ms`。 | Android 真机 `ce8c42ac` + `adb shell am start -W` |
| 热启动/重新打开 | 待定 | 已记录历史热启动 `TotalTime=64ms`。 | Android 真机 |
| 登录到首页耗时 | 待定 | 未形成完整记录。 | 测试账号真机 |
| 会话列表首屏 | 待定 | 单机消息首页可见；未做 50/500/2000 会话数据压测。 | 50/500/2000 会话数据 |
| 聊天页首屏 | 待定 | 未做 50/500/2000 消息数据压测。 | 50/500/2000 消息数据 |
| 消息发送耗时 | 待定 | 文本待发队列和重连 flush 有代码证据；真实弱网耗时未记录。 | 正常网/弱网 |
| 图片加载和缓存 | 待定 | 已优化头像后台读取、预热、并发去重和尺寸解码；多图片会话未压测。 | 多图片会话 |
| 内存占用 | 待定 | release 冷启动后 PSS 约 `239MB`，历史主 Tab 切换后 PSS 约 `290MB`；本轮 debug 包启动后 `TOTAL PSS=381719KB`、`TOTAL RSS=569828KB`，仅作为调试构建参考。 | `adb shell dumpsys meminfo com.lpp.mobile` |
| WebSocket/Gateway 重连 | 待定 | 断网提示和恢复隐藏已真机验证；Gateway 重连和 `/sync` 有代码/测试证据。 | 断网/切网 |
| 通话建立耗时 | 待定 | 未记录 | 双真机 |
| 通话稳定性 | 待定 | 未记录 | 双真机 + 弱网 |
| 推送到达和点击 | 待定 | 未记录 | 前台/后台/杀进程 |

## 3. 稳定性场景

| 场景 | 必测内容 | 当前状态 |
| --- | --- | --- |
| 弱网 | 登录、消息发送、图片上传、客服接管、审批操作。 | 部分执行：系统断网提示和恢复隐藏已验证；限速、蜂窝、弱网发送耗时未执行。 |
| 断网恢复 | Gateway 重连、增量同步、失败消息重试。 | 部分执行：恢复网络后消息首页继续显示；离线消息补齐和待发消息自动重发未执行。 |
| 多端并发 | 好友申请、群禁言、客服接管、会话关闭。 | 未执行 |
| 长列表 | 会话列表、聊天消息、客户列表、群成员列表。 | 代码已做懒加载、缓存和防抖优化；真实大数据压测未执行。 |
| 后台前台切换 | 会话刷新、通话、推送、权限恢复。 | 未执行 |
| Android 厂商差异 | Vivo、OPPO、小米、华为、Pixel。 | 未执行 |

## 4. App 侧性能待办（不含服务端缺接口）

| 编号 | 优先级 | 待办 | 验证方法 |
| --- | --- | --- | --- |
| PERF-APP-001 | P0 | 建立 Android P0 路径耗时记录：启动、登录、进首页、进聊天、发送文本、退出登录。 | 真机 + `adb shell am start -W` + App 诊断日志。 |
| PERF-APP-002 | P0 | 验证断网发送文本和恢复后的自动重发。 | 真机关闭网络发送消息，再恢复网络，记录状态变化和耗时。 |
| PERF-APP-003 | P1 | 会话列表、聊天消息、客户列表、群成员列表大数据滚动压测。 | 500/1000/2000 条数据，记录帧率、掉帧、内存。 |
| PERF-APP-004 | P1 | 头像和图片密集场景压测。 | 多头像联系人、多图片聊天记录，记录内存峰值和滚动流畅度。 |
| PERF-APP-005 | P1 | 后台 10 分钟、30 分钟、2 小时后恢复。 | 切后台后恢复，检查 Gateway、会话刷新、未读和权限状态。 |
| PERF-APP-006 | P1 | 推送和后台来电专项。 | 前台、后台、杀进程、锁屏状态下验证到达和点击跳转。 |
| PERF-APP-007 | P2 | 低端 Android 设备复测。 | 低内存设备上重复启动、长列表和图片密集场景。 |
| PERF-APP-008 | P2 | 错误诊断增强。 | 已完成 breadcrumbs、sessionId、clientTraceId、HTTP `X-Client-Trace-Id`、本地诊断包复制/导出；后续验证服务端落库和告警。 |

## 7. 当前阻塞

- 没有标准测试数据规模。
- 没有版本间性能对比方式。
- 没有大数据长列表、低端机、通话和推送专项性能数据。
- 没有弱网发送耗时和后台长稳数据。
- debug 包内存只能作为调试参考，正式交付仍以 release 包和标准数据规模复测为准。
