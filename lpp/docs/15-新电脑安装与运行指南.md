# LPP 新电脑安装与运行指南

更新时间：2026-05-22

适用对象：另一台电脑上的 Codex、开发人员或测试人员。

目标：在一台新电脑上把当前 LPP 项目跑起来，并让 Codex 继续按本项目 ECC 流程工作。

## 1. 重要结论

当前项目已经自带项目级 ECC 内容，不需要在新电脑重复安装到项目里。

这里的“项目已经自带”指的是：这些 ECC 文档、项目规则和本地 skill 文件本身就是项目目录中的普通文件。只要把整个 `lpp-flutte` 项目复制到新电脑，或者从 git 仓库 clone 下来，它们就会一起到新电脑。

需要区分两件事：

| 事项 | 是否需要在新电脑安装 |
| --- | --- |
| Codex App / Codex CLI 这个工具本身 | 需要。如果新电脑还没有 Codex，需要先安装并登录。 |
| LPP 项目级 ECC 文档和规则 | 不需要单独安装。它们已经在项目目录里，跟代码一起走。 |
| LPP 项目本地 `.codex/skills/*.md` | 不需要单独安装。它们已经在项目目录里。 |
| 全局 Codex skills | 可选。只有你希望新电脑上的所有项目都使用同一套通用 skills 时才安装。 |

已随项目提供的内容：

| 内容 | 项目路径 | 新电脑处理 |
| --- | --- | --- |
| ECC 工作流文档 | `lpp/docs/00-ECC工作流.md` | 保留，不重复创建 |
| 需求规格说明书 | `lpp/docs/01-需求规格说明书.md` | 保留 |
| 功能矩阵 | `lpp/docs/02-功能矩阵和实现情况.md` | 保留 |
| 技术方案 | `lpp/docs/03-技术方案.md` | 保留 |
| 接口与功能映射 | `lpp/docs/05-接口与服务端依赖.md`、`lpp/docs/14-功能接口映射表.md` | 保留 |
| Codex 项目规则 | `lpp/lpp_mobile/AGENTS.md` | 保留 |
| 项目本地 skills | `.codex/skills/` | 保留 |
| 自动化脚本 | `lpp/scripts/` | 保留 |

新电脑只需要补齐运行环境：

- Flutter SDK
- Android Studio / Android SDK
- Android 真机调试环境
- 可选：Codex App 或 Codex CLI

## 2. 项目目录要求

把整个项目目录放到新电脑，例如：

```bash
/Users/你的用户名/Development/lpp-flutte
```

项目根目录应包含：

```text
.codex/
lpp/
lpp/README.md
lpp/docs/
lpp/scripts/
lpp/lpp_mobile/
```

Flutter 项目根目录是：

```bash
lpp/lpp_mobile
```

不要在 `lpp-flutte` 或 `lpp` 目录直接执行 Flutter 构建命令；必须进入 `lpp/lpp_mobile`。

## 3. 不需要复制或提交的目录

以下目录是构建缓存或本机环境产物。没有也正常，新电脑运行后会重新生成。

```text
lpp/lpp_mobile/build/
lpp/lpp_mobile/.dart_tool/
lpp/lpp_mobile/android/.gradle/
lpp/lpp_mobile/ios/Pods/
lpp/lpp_mobile/macos/Pods/
```

如果这些目录不存在，直接忽略。

## 4. 安装 Flutter 环境

### 4.1 安装 Flutter SDK

安装完成后执行：

```bash
flutter --version
flutter doctor -v
```

如果 `flutter` 命令不存在，需要把 Flutter SDK 的 `bin` 目录加入 PATH。

### 4.2 安装 Android Studio 和 Android SDK

安装 Android Studio，并确认包含：

- Android SDK
- Android SDK Platform-Tools
- Android SDK Build-Tools
- Android SDK Command-line Tools
- Android Emulator，可选
- JDK，Android Studio 通常自带

然后执行：

```bash
flutter doctor --android-licenses
flutter doctor -v
```

按提示接受 Android licenses。

## 5. 连接 Android 手机

手机端开启：

- 开发者模式
- USB 调试
- 信任当前电脑

电脑端执行：

```bash
flutter devices
```

能看到设备后，继续下一步。

如果看不到设备，先检查：

- USB 线是否支持数据传输。
- 手机是否弹出并确认 USB 调试授权。
- Android Studio 的 platform-tools 是否安装。
- macOS 是否需要重新插拔或重启 adb。

可尝试：

```bash
adb devices
```

## 6. 安装项目依赖

进入 Flutter 项目根目录：

```bash
cd /你的路径/lpp-flutte/lpp/lpp_mobile
```

拉取依赖：

```bash
flutter pub get
```

如果依赖拉取失败，先检查：

- 网络是否可访问 pub.dev。
- Flutter SDK 版本是否过旧。
- 当前目录是否存在 `pubspec.yaml`。

## 7. 运行到手机

在 `lpp/lpp_mobile` 目录执行：

```bash
flutter run
```

如果有多个设备：

```bash
flutter devices
flutter run -d 设备ID
```

常见错误：

| 错误 | 处理 |
| --- | --- |
| `No pubspec.yaml file found` | 当前目录错了，进入 `lpp/lpp_mobile` 后再执行。 |
| `ADB exited with exit code 1` | 检查手机授权、存储空间、是否已有旧包冲突；必要时卸载旧 App 后重试。 |
| `Android license status unknown` | 执行 `flutter doctor --android-licenses`。 |
| 找不到设备 | 执行 `flutter devices` 和 `adb devices`，检查 USB 调试。 |

## 8. Codex 在新电脑上的使用方式

如果新电脑还没有安装 Codex，需要先安装 Codex App 或 Codex CLI，并登录你的账号。安装 Codex 是为了让新电脑具备 AI 协作开发能力；它不是安装 LPP 项目的 ECC。

安装并登录 Codex 后，打开项目根目录：

```bash
/你的路径/lpp-flutte
```

不要只打开 `lpp/lpp_mobile`，否则 Codex 可能看不到项目根部的 `.codex/skills/` 和完整 `lpp/docs/`。

Codex 首次接手时应先读取：

```text
lpp/lpp_mobile/AGENTS.md
lpp/docs/README.md
lpp/docs/00-ECC工作流.md
lpp/docs/01-需求规格说明书.md
lpp/docs/02-功能矩阵和实现情况.md
lpp/docs/03-技术方案.md
```

如果 Codex 没有自动读取项目规则，可以直接把下面这句话发给它：

```text
请先阅读 lpp/lpp_mobile/AGENTS.md 和 lpp/docs/README.md。本项目已经自带项目级 ECC，请按 lpp/docs/00-ECC工作流.md 执行。
```

涉及接口排查时读取：

```text
lpp/docs/05-接口与服务端依赖.md
lpp/docs/14-功能接口映射表.md
```

涉及测试和发布时读取：

```text
lpp/docs/04-测试验收方案.md
lpp/docs/06-发布风险与检查清单.md
lpp/docs/12-版本交付报告.md
lpp/docs/13-自动化执行脚本说明.md
```

## 9. ECC 不需要重复安装的判断

在新电脑项目根目录执行：

```bash
find .codex/skills -maxdepth 1 -type f | sort
find lpp/docs -maxdepth 1 -type f | sort
test -f lpp/lpp_mobile/AGENTS.md && echo "AGENTS.md exists"
```

如果能看到 `.codex/skills/*.md`、`lpp/docs/*.md` 和 `lpp/lpp_mobile/AGENTS.md`，说明项目级 ECC 已经存在。

不要重复执行全局 ECC 安装，除非明确要给新电脑的所有项目使用通用 skills。

如果新电脑没有 Codex，则需要安装的是 Codex 工具本身，不是重新安装项目 ECC。安装后只要用 Codex 打开 `/你的路径/lpp-flutte`，再让它读取 `lpp/lpp_mobile/AGENTS.md` 和 `lpp/docs/README.md` 即可。

## 10. 文档检查

在项目根目录执行：

```bash
cd /你的路径/lpp-flutte
bash lpp/scripts/docs/check-docs.sh
```

期望结果：

```text
missing_count=0
Docs check completed.
```

如果出现 missing link，需要优先修复文档链接。

## 11. 自动化测试

进入 Flutter 项目根目录：

```bash
cd /你的路径/lpp-flutte/lpp/lpp_mobile
```

执行全部 Flutter 测试：

```bash
flutter test ../scripts/mobile/test/flutter
```

也可以执行脚本入口：

```bash
cd /你的路径/lpp-flutte
bash lpp/scripts/test/run-flutter-checks.sh
```

当前项目文档记录过：此前自动化测试存在 admin token 相关失败。因此新电脑测试失败时，不要直接判断为环境问题，需要查看失败用例和日志。

## 12. 推荐首次验证顺序

新电脑首次运行建议按顺序执行：

```bash
cd /你的路径/lpp-flutte
bash lpp/scripts/docs/check-docs.sh

cd lpp/lpp_mobile
flutter doctor -v
flutter pub get
flutter devices
flutter run
```

运行成功后，再考虑：

```bash
flutter analyze
flutter test ../scripts/mobile/test/flutter
```
