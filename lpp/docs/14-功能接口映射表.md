# LPP 功能接口映射表

更新时间：2026-05-26

最后核对方式：基于当前 Flutter 代码中的路由、datasource、provider、页面直接调用和接口依赖文档静态整理。本文档用于排查问题时从“功能”快速定位“服务端接口、token 域、客户端代码证据和排查重点”。代码中存在调用不代表接口已全部联调通过。

## 1. 使用方式

排查功能问题时按以下顺序使用：

1. 在本文档按功能名定位接口。
2. 确认接口域和 token 是否正确。
3. 对照客户端代码证据定位 datasource、provider 或页面。
4. 记录 requestId、status、服务端 code、message 和 response body。
5. 如涉及权限、角色、空间或 admin API，同时检查 [安全权限与数据治理](09-安全权限与数据治理.md)。

## 2. 接口域速查

| 接口域 | Token | 主要功能 |
| --- | --- | --- |
| `/api/platform/v1/*` | platform token | 平台登录、空间、租户、设备、admin token 签发。 |
| `/api/client/v1/*` | tenant access token | IM、好友、群、客户侧客服、企业空间、资料、设置。 |
| `/api/admin/v1/*` | admin access token | 管理员/所有者工作台、客户池、客服中心、审计、治理。 |
| `/ws/client` | tenant access token | 会话、消息、在线状态等实时事件。 |
| `/hubs/voicecall` | tenant access token | 音视频通话信令。 |
| `/api/v1/notifications/*` | tenant access token 或通知服务约定 | 推送设备注册。 |

## 12. 常见问题排查入口

| 问题表现 | 优先检查接口 |
| --- | --- |
| 登录失败 | `POST /api/platform/v1/auth/login`、captcha 相关接口。 |
| 企业空间进不去 | `GET /api/platform/v1/my/tenants`、`POST /api/platform/v1/auth/select-tenant`。 |
| 管理工作台 401/403 | `POST /api/platform/v1/auth/admin-token`、对应 `/api/admin/v1/*` 接口 token。 |
| 企业头像后台更新后客户端没变 | `GET /api/client/v1/tenant/info`、空间缓存、头像 URL 缓存。 |
| 好友申请重复或处理失败 | `GET /api/client/v1/friends/requests`、`POST /api/client/v1/friends/requests/{requestId}/handle`。 |
| 扫码加好友后还要再点接受 | `POST /api/client/v1/friends/invite-qr/{token}/accept` 返回的 requestId 和申请状态刷新。 |
| 消息显示发送成功但实际失败 | direct/group/customer-service 发送接口 response、Gateway 同步事件。 |
| 搜索结果不能跳原消息 | `GET /api/client/v1/search/messages` 返回的 conversationId/messageId/seq。 |
| 快捷翻译显示暂无消息 | 当前会话是否存在对方文本消息、`POST /api/client/v1/translate/message`。 |
| 管理员看到的会话不是客服客户会话 | `GET /api/admin/v1/customer-service/center/threads` 的 `threadType` 和 UI 命名。 |
| 客户标签筛选无效 | `GET /api/admin/v1/users?tags=...&tagMatch=any`、`PUT /api/admin/v1/users/{id}/tags`。 |
| 群禁言不生效 | `PUT /api/client/v1/groups/{groupId}/mute-mode`、`PUT /members/{userId}/mute`、发送消息接口 403/409。 |
| 通话无法建立 | `POST /api/client/v1/voicecall/sessions`、`/hubs/voicecall`、系统权限。 |
| 推送收不到 | `/api/v1/notifications/devices` 注册结果、厂商通道 token、本地通知权限。 |
