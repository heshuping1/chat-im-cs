# 服务端支持

更新时间：2026-05-25

本文只保留当前仍需要服务端补齐、修复、确认或提供配置的事项。已经在当前接口文档中明确支持的能力，不再作为缺口列出，例如：客服本人接待历史 `staff/service-history`、客户维度服务历史、客服接待状态 `reception/status`、客户/会话管理分页、企业群发、企业公告管理与已读统计、翻译接口、客户画像 `profile-card`、客户端错误上报、群冻结/禁言/移除/解散、管理端客服状态、知识库与 AI 建议接口。

## 处理口径

| 问题类型 | 含义 |
| --- | --- |
| 接口不可用/待修复 | 接口已存在或已文档化，但真实账号调用失败、权限不通或返回不稳定。 |
| 缺失接口 | 当前接口文档未提供正式能力，客户端只能空态、占位或隐藏入口。 |
| 字段/事件缺失 | 接口存在，但缺关键字段、统计、实时事件或 Push 字段。 |
| 业务口径待确认 | 接口存在，但真实业务副作用、权限边界、状态流转或错误码需要服务端明确。 |
| 配置/环境待确认 | 需要服务端提供配置、三方通道、联调环境或健康检查。 |

## 1. 登录、验证码和空间未读

### 1.1 验证码生产通道

客户端已经具备邮箱、短信和图形验证码输入能力。服务端仍需要确认生产链路配置：

| 事项 | 服务端确认点 |
| --- | --- |
| 邮箱验证码 | 发送通道、模板、有效期、重发冷却、频控、错误码、送达率监控。 |
| 短信验证码 | 供应商、国家区号、签名、模板、有效期、重发冷却、频控、到达率。 |
| 图形验证码 | 触发条件、题型字段、过期时间、错误码、失败重试、是否允许自动计算题。 |

### 1.2 空间未读汇总

客户端当前空间未读已经按普通 IM 可见会话计算；跨空间仍依赖平台汇总接口。

| 能力 | 服务端确认点 |
| --- | --- |
| 跨空间汇总 | `/api/platform/v1/my/spaces/unread-summary` 只统计普通 IM 的 `direct/group`，不统计在线客服临时会话、工作台、公告和系统事件。 |
| 字段语义 | `unreadConversationCount` 表示有普通 IM 未读的会话数，`unreadMessageCount` 表示普通 IM 未读消息总数。 |
| 实时事件 | `space.notice` 指向的目标空间需要和 `unread-summary` 同口径。 |

### 2.2 管理端客服运营统计

| 能力 | 服务端诉求 |

| 账号注销/取消注销 | 冷静期、重新登录撤销、数据保留/清除、企业关系清理和错误码。 |
| 隐私设置 | 手机号/LPP 搜索、好友申请、资料可见性等真实副作用。 |
| PC 个人偏好同步 | 如果要跨端同步语言、时区、自动翻译、显示偏好、聊天发送习惯、线路选择，需要统一设置读写接口；否则 PC 本地保存。 |

## 6. 提交服务端时的验收口径

| 类别 | 验收标准 |
| --- | --- |
| 接口可用 | 真实角色账号可调用，权限错误有稳定 code。 |
| 字段稳定 | 文档字段、真实返回字段和客户端解析一致；缺值有明确空态。 |
| 实时刷新 | 关键状态变化有 Gateway 或 Push 事件，客户端不靠轮询猜测。 |
| 业务副作用 | 权限、冻结、踢出、隐私、状态切换等会真实影响业务链路。 |
| 错误可诊断 | 返回 `requestId/traceId`，客户端日志可提供给服务端排查。 |
| 多角色一致 | 所有者、管理员、客服、普通用户看到的入口、字段和权限一致。 |
