# LPP 代码现状与证据索引

更新时间：2026-05-22

最后核对方式：静态盘点代码目录、路由、endpoint 和测试文件。本文档用于支撑 [功能矩阵和实现情况](02-功能矩阵和实现情况.md) 中的状态判断。

## 1. 状态证据口径

| 证据类型 | 说明 |
| --- | --- |
| 代码证据 | 能定位到具体页面、provider、datasource、service、model、controller 或测试文件。 |
| 接口证据 | 能定位到具体 endpoint、method、参数和返回模型。 |
| 测试证据 | 有自动化测试、接口联调记录或真机验收记录。 |
| 产品证据 | 需求规格和交互说明中有明确验收标准。 |

## 2. 功能状态判定规则

| 功能矩阵状态 | 最低证据要求 |
| --- | --- |
| 可用 | 需求明确；代码证据明确；接口或本地逻辑明确；至少有自动化、接口联调或真机验收之一。 |
| 部分可用 | 需求明确；代码证据明确；已完成部分和未完成缺口明确。 |
| 未实现 | 需求存在但没有可用入口或只有占位；不得写成部分可用。 |
| 需服务端 | 客户端代码无法单独完成；必须能关联服务端接口、字段、事件或权限依赖。 |
| 需真机验证 | 自动化无法覆盖；必须在真机验收前保持未验收状态。 |
| 已验收 | 必须有版本、设备、账号、执行方式和结果记录。 |

## 5. 接口证据

接口清单见 [接口与服务端依赖](05-接口与服务端依赖.md)。按功能排查接口时见 [功能接口映射表](14-功能接口映射表.md)。当前已经从代码中识别到以下关键接口域：

- `/api/platform/v1/*`
- `/api/client/v1/*`
- `/api/admin/v1/*`
- `/ws/client`
- `/hubs/voicecall`

需要注意：

- 识别到代码调用不等于已联调通过。
- 接口返回字段和服务端权限仍需要真实环境确认。
- admin API 必须额外检查 token 域。
