# LPP 自动化执行脚本说明

## 1. 文档目的

本文档用于说明 LPP 项目中可执行脚本的存放位置、执行目录、使用场景、输出结果和风险边界。

脚本负责执行，文档负责说明，执行结果回填到测试验收、缺口清单和版本交付报告。

## 2. 脚本目录规则

| 目录 | 用途 |
| --- | --- |
| `lpp/scripts/docs` | 文档体检、链接检查、文档状态扫描。 |
| `lpp/scripts/test` | 项目级测试、静态分析、P0 smoke、真机辅助检查。 |
| `lpp/scripts/release` | 版本交付信息收集、报告辅助生成、发布前检查。 |
| `lpp/scripts/mobile` | 移动端已有专项脚本，包括 API 调试、测试数据、平台测试和 e2e。 |
| `lpp/scripts/pc` | PC 客服客户端专项脚本，包括 Electron/React 类型检查、构建和浏览器自动化。 |

新增项目级自动化脚本优先放在 `lpp/scripts`。只服务 Flutter 移动端内部调试的脚本可以继续放在 `lpp/scripts/mobile`。

## 3. 脚本命名规则

- 使用小写和短横线，例如 `check-docs.sh`。
- 脚本名体现动作和对象，例如 `run-flutter-checks.sh`。
- 会修改数据、创建账号、审批申请、重置状态的脚本，文件名必须体现风险，例如 `reset-test-accounts.sh`。
- 脚本开头必须说明执行目录、用途、是否会写文件、是否会请求真实接口。

## 4. 推荐脚本清单

| 脚本 | 目录 | 用途 | 是否已创建 |
| --- | --- | --- | --- |
| `check-docs.sh` | `lpp/scripts/docs` | 检查文档清单、坏链、未闭合标记。 | 是 |
| `run-flutter-checks.sh` | `lpp/scripts/test` | 执行 `flutter analyze` 和 `flutter test ../scripts/mobile/test/flutter`。 | 是 |
| `run_full_regression.sh` | `lpp/scripts/mobile/test` | 移动端完整可重复回归：文档检查、pub get、静态分析、全量 Flutter 测试、Android 构建、可选安装启动，并输出报告。 | 是 |
| `run_pc_browser_tests.sh` | `lpp/scripts/pc/test` | PC 客服客户端完整本地回归：`npm run typecheck`、`npm run build`、`npm run test:browser`。 | 是 |
| `run-p0-smoke.sh` | `lpp/scripts/test` | 辅助执行 Android P0 smoke 检查。 | 否 |
| `collect-release-report.sh` | `lpp/scripts/release` | 收集版本、测试、文档体检和构建信息。 | 是 |

## 6. 执行目录规范

| 脚本类型 | 推荐执行目录 |
| --- | --- |
| 文档体检 | `/Users/treesoft/Downloads/lpp-flutte/lpp` |
| 项目级测试 | `/Users/treesoft/Downloads/lpp-flutte/lpp` |
| Flutter 命令 | `/Users/treesoft/Downloads/lpp-flutte/lpp/lpp_mobile` |
| Android 安装和运行 | `/Users/treesoft/Downloads/lpp-flutte/lpp/lpp_mobile` |
| PC 客服客户端测试 | `/Users/treesoft/Downloads/lpp-flutte/lpp` |
| 测试数据脚本 | 以脚本头部说明为准 |

完整可重复回归推荐命令：

```bash
cd /Users/treesoft/Downloads/lpp-flutte/lpp
scripts/mobile/test/run_full_regression.sh --repeat=1
```

重复压测命令：

```bash
scripts/mobile/test/run_full_regression.sh --repeat=3 --continue-on-fail
```

脚本默认报告目录为 `reports/mobile/full-regression/<timestamp>/`，交付前应把 `summary.md` 中的结果回填到版本交付报告。

PC 客服客户端可重复回归命令：

```bash
cd /Users/treesoft/Downloads/lpp-flutte/lpp
scripts/pc/test/run_pc_browser_tests.sh
```

只跑浏览器冒烟或透传 Playwright 参数：

```bash
scripts/pc/test/run_pc_browser_tests.sh --no-typecheck --no-build --headed
```

PC 脚本默认报告为 `lpp_pc_client/playwright-report/index.html`。该脚本只做本地类型检查、构建和浏览器自动化，不请求业务接口、不创建测试数据。

## 8. 安全边界

- 默认脚本只做本地检查。
- 会请求真实接口的脚本必须在说明中标记。
- 会创建、审批、删除、重置数据的脚本必须在执行前明确环境和账号。
- 脚本输出不得打印 token、cookie、密码、验证码。
- 脚本失败不得自动清理业务数据，除非脚本名和说明明确声明。
