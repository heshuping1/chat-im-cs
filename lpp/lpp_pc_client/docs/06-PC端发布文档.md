# PC 端发布文档

状态：当前有效

日期：2026-06-06

适用范围：`lpp_pc_client`

## 1. 目的

本文档是 PC 端发布的唯一入口，用于串联发布范围、发布前检查、试用验收、反馈闭环、服务端依赖、客户端更新合同和回滚判断。

后续发布讨论不要直接从零散文档开始，应先读取本文档，再按发布阶段跳转到对应资料。

## 2. 当前发布口径

当前发布口径以 PC 普通 IM 为主，发布候选冻结、回归命令、人工检查和暂停条件以 [release/01-PC普通IM发布候选检查.md](release/01-PC普通IM发布候选检查.md) 为准。

如后续新增在线客服、通讯录、设置、更新服务或安装包发布范围，必须先在本文档登记，再同步更新需求、功能矩阵和对应技术方案。

## 4. 发布前闭环流程

### 4.1 范围冻结

1. 读取 [02-PC端交付物总览.md](02-PC端交付物总览.md)，确认本次发布包含范围和不包含范围。
2. 读取 [01-PC端需求规格说明.md](01-PC端需求规格说明.md)，确认需求口径没有新增未登记项。
3. 读取 [03-PC端功能矩阵.md](03-PC端功能矩阵.md)，确认本次发布功能均有实现状态、代码入口、测试入口和风险说明。
4. 如果发布范围影响模块边界、数据流或接口策略，同步读取 [04-PC端技术方案总览.md](04-PC端技术方案总览.md) 和 `technical/` 下对应详细方案。

### 4.2 依赖确认

1. 读取 [05-服务端支持.md](05-服务端支持.md)，确认服务端缺口是否阻断本次发布。
2. 如果涉及客户端更新、强制更新或版本检测，读取 [release/05-client-update-service-contract.md](release/05-client-update-service-contract.md)。
3. 如果涉及图标、安装包图标或品牌资产，读取 [release/04-PC图标统一管理手册.md](release/04-PC图标统一管理手册.md)。

### 4.3 回归与人工验收

1. 按 [release/01-PC普通IM发布候选检查.md](release/01-PC普通IM发布候选检查.md) 执行必跑回归命令。
2. 按 [release/01-PC普通IM发布候选检查.md](release/01-PC普通IM发布候选检查.md) 执行人工检查清单。
3. 如需要真实用户试用，按 [release/02-PC普通IM试用说明.md](release/02-PC普通IM试用说明.md) 执行。
4. 试用或验收问题统一记录到 [release/03-PC普通IM试用反馈闭环.md](release/03-PC普通IM试用反馈闭环.md)。

### 4.4 发布判断

满足以下条件才可以进入发布：

1. 本次发布范围已冻结。
2. 必跑回归命令通过。
3. 人工验收通过。
4. 没有未关闭的 P0/P1。
5. 服务端缺口已确认不阻断本次发布，或已完成闭环。
6. 试用问题已按等级关闭、延期或明确不阻断。
7. 回滚或暂停发布条件已确认。

## 5. 暂停发布条件

出现以下任一情况，应暂停发布并回到问题闭环：

1. 发布范围无法确认，或需求口径与实现不一致。
2. 必跑回归命令失败且没有完成修复和复测。
3. 人工验收出现 P0/P1。
4. 服务端接口、字段或 Gateway 行为变化导致核心链路不可用。
5. 消息收发、未读/已读、媒体发送、历史查看、批量操作等核心能力出现阻断。
6. 客户端更新策略可能导致用户无法启动、无法登录或无法恢复到稳定版本。
7. 回滚方案不可执行。

每次发布应在发布讨论或发布记录中保留以下信息：

```text
不包含范围：
回归命令：
人工验收结论：
服务端缺口结论：
试用反馈结论：
是否涉及客户端更新：
是否涉及图标/品牌资产：
暂停发布条件复核：
回滚方案：
最终判断：
```

## 7. 后续新增发布范围

新增发布范围时，按以下顺序更新文档：

1. 在本文档登记发布入口和发布资料路由。
2. 在 [01-PC端需求规格说明.md](01-PC端需求规格说明.md) 登记需求。
3. 在 [03-PC端功能矩阵.md](03-PC端功能矩阵.md) 更新功能状态、代码入口、测试入口和风险。
4. 如涉及服务端依赖，更新 [05-服务端支持.md](05-服务端支持.md)。
5. 如影响技术方案，更新 [04-PC端技术方案总览.md](04-PC端技术方案总览.md) 或 `technical/` 对应详细方案。
6. 如进入发布候选，补充对应发布候选检查或专项发布检查。
