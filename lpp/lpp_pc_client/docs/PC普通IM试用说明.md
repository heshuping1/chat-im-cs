# PC 普通 IM 试用说明

本文档用于 PC 普通 IM 小范围真实客服试用。试用范围只包含 PC 端普通 IM，不包含 App、在线客服临时会话、主题系统或管理后台能力。

## 可试用能力

- 会话收发：支持私聊、群聊的 push、pull、history、发送后本地回显统一解析。
- 未读/已读：采用微信式规则；未打开会话收到消息只增加未读，点击会话后按 `lastMessageSeq` 标记已读，当前已打开会话实时新消息可直接已读。
- 新消息提示：点击有未读的会话后，聊天区显示“↑↑ N 条新消息”，可跳到本地已加载范围内第一条未读。
- 文本和表情：支持真实 emoji 输入和显示，不显示 `[笑脸]` 这类占位文本。
- 图片：支持按钮选择、拖拽、粘贴发送；支持上传中/失败提示、加载过程图、点击预览、右键下载。
- 文件：支持按钮选择、拖拽发送；展示文件名和大小，支持下载。
- 名片：名片消息展示为卡片，点击可查看资料浮层。
- 群聊：群 sender 优先展示成员名，不展示 UUID；群成员事件按系统提示展示；群资料支持成员列表和基础信息。
- 消息操作：支持回复、复制、转发、多选、撤回、删除、收藏、翻译；右键菜单按消息类型显示合理操作。
- 历史查看：支持当前已加载消息的搜索、类型筛选、结果跳转、空状态。
- 会话管理：支持全部/未读/好友/群聊筛选、最近消息预览、草稿保留、置顶排序读取、本地 session 级免打扰/隐藏。
- 服务端一致性：收藏、撤回、删除、转发后会触发 refetch，避免状态闪回。
- 批量操作：批量转发和批量删除按单条接口顺序执行，部分失败时给出成功/失败数量提示。

## 已知限制

- 会话置顶：PC 只读取服务端返回的 `isPinned` 参与排序，暂不支持 PC 端设置/取消置顶。
- 免打扰：PC 只读取服务端返回的 `isMuted`；右键免打扰为本地 session 级提示，刷新或重新登录后不保证保留。
- 删除/隐藏会话：当前为本地 session 级隐藏；服务端删除、隐藏、恢复语义未闭环。
- 群公告：暂无群公告读取接口，群资料面板显示明确空状态。
- 群成员角色/群昵称：优先使用服务端返回的 `displayName`、`role`、`memberRole`；更细的群昵称、群主、管理员字段依赖服务端稳定返回。
- @ mentions：PC 已按现有群消息发送接口提交 `mentions`，但 push/pull/history 回显字段和“有人 @ 我”提醒规则仍需服务端确认。
- 回复引用：PC 能展示回显里携带的引用摘要；如果历史消息只返回 `replyToMessageId` 而没有摘要，无法凭空恢复引用内容。
- 批量操作：当前按单条接口顺序执行，不具备服务端批量原子性。
- 历史范围：历史搜索和类型筛选只覆盖当前已加载消息；第一条未读不在本地已加载范围时，会提示继续加载历史消息。

## 试用账号和环境注意事项

- 使用测试或灰度租户账号，不要在文档、截图、聊天消息或反馈里写真实密码。
- 确认 PC 端连接的是预期 API 环境和 WebSocket 网关。
- 试用前确认账号至少具备：一个好友私聊、一个群聊、可发送图片/文件的媒体上传权限。
- 如果要验证多端未读，请准备另一个客户端账号或 App 账号发送消息。
- 截图反馈时注意遮挡客户手机号、邮箱、证件、资金、订单等敏感信息。

## 试用脚本

### 私聊

1. 从另一端给 PC 当前账号发送文本消息。
2. 不点击会话，确认会话列表出现未读红点。
3. 点击会话，确认红点清掉，聊天区出现“↑↑ N 条新消息”。
4. 点击新消息提示，确认跳到第一条未读或出现继续加载历史提示。
5. 回复该消息，确认引用关系显示正常。
6. 右键文本消息，依次验证复制、翻译、收藏、转发、删除。
7. 发送一条自己的消息，确认不产生未读。
8. 撤回自己刚发送的消息，刷新或等待 refetch 后确认仍显示“消息已撤回”。

### 群聊

1. 进入群聊，确认消息 sender 显示姓名或可读昵称，不显示 UUID。
2. 让群成员发送文本消息，确认 PC 收到后按当前会话规则已读或按非当前会话规则增加未读。
3. 输入 `@` 选择群成员并发送，确认正文和本地回显正常。
4. 验证群成员加入等事件显示为系统提示，不进入普通气泡。
5. 打开群资料，查看群名称、成员数、成员列表、角色显示和群公告空状态。

### 图片和文件

1. 点击“图片”选择一张图片，确认出现待发送附件。
2. 点击发送，确认上传中状态、发送后图片展示、点击可预览。
3. 右键图片，确认可下载、多选、撤回、删除等合理项。
4. 拖拽或粘贴图片到输入区，确认可发送。
5. 点击“文件”选择 PDF 或普通文件，确认展示文件名和大小。
6. 断网或使用异常文件模拟上传失败，确认附件不丢失且有明确失败提示。

### 历史搜索

1. 打开聊天历史。
2. 搜索一个文本关键词，确认结果数量和跳转可用。
3. 按类型筛选文字、图片、文件、收藏。
4. 点击结果，确认能定位到当前已加载消息。
5. 搜索不存在的关键词，确认空状态清晰。

### 会话未读和草稿

1. 在会话 A 输入草稿，不发送。
2. 切换到会话 B，再回到会话 A，确认草稿仍在。
3. 在会话 A 未打开时从另一端发送消息，确认未读红点保留。
4. 打开会话 A，确认立即标记已读并显示新消息提示。
5. 当前打开会话 A 时从另一端发送消息，确认消息直接展示且不产生未读红点。

### 批量操作

1. 右键消息进入多选。
2. 选择多条文本/图片/文件消息。
3. 批量转发到另一个会话，确认目标会话能看到成功转发的消息。
4. 模拟部分失败时，确认提示包含成功数量和失败数量。
5. 批量删除消息，确认危险操作有确认，部分失败有明确提示。

## 试用前高风险项检查

当前专项测试已覆盖以下高风险闭环：

- push/pull 非当前会话只增加未读，不自动已读。
- 当前打开会话收到实时消息可直接已读，不显示未读红点。
- 点击会话后清红点并显示“↑↑ N 条新消息”。
- text、emoji、image、file、contact card、event、不支持类型都有展示兜底。
- 图片/文件发送、拖拽、粘贴、上传失败提示。
- 名片点击查看资料。
- 撤回、删除、收藏、转发 refetch 后状态一致。
- 批量转发、批量删除部分失败有明确提示。

如试用中出现误已读、红点清错、收到消息不显示、消息空白、图片/文件失败无提示、群 sender 显示 UUID、批量操作静默失败，应优先按问题反馈模板提交并暂停扩大试用范围。
