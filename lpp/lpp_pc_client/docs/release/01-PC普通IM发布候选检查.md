# PC 普通 IM 发布候选检查

本文档用于 PC 普通 IM 进入发布候选冻结前的范围确认、风险复盘和回归检查。范围只包含 PC 普通 IM，不包含 App、在线客服临时会话、主题系统或管理后台能力。

## 试用反馈复盘

### 已关闭问题

- 当前仓库未维护独立试用问题列表；截至本次发布候选检查，文档中没有未关闭的实际 P0/P1 试用反馈记录。
- 已通过专项测试覆盖并回归的核心闭环包括：
  - push/pull/history/local echo normalize。
  - 微信式未读/已读。
  - emoji 真表情。
  - 图片/文件发送、上传失败、预览、下载。
  - 名片点击查看。
  - 群事件系统提示。
  - 回复、复制、转发、多选、撤回、删除、收藏、翻译。
  - 历史搜索、类型筛选、结果跳转。
  - 草稿不丢。
  - 群 sender 不显示 UUID。
  - 收藏、撤回、删除、转发 refetch 后状态一致。
  - 批量转发、批量删除部分失败有明确提示。

### 未关闭问题

- 当前没有记录中的未关闭 P0。
- 当前没有记录中的阻断性 P1。
- 后续如出现未读误清、消息丢失、消息空白、图片/文件阻断、refetch 回退、批量操作静默失败，应暂停发布候选推进并按 [03-PC普通IM试用反馈闭环.md](03-PC普通IM试用反馈闭环.md) 处理。

### 服务端缺口

详见 [05-服务端支持.md](../05-服务端支持.md)。当前仍未闭环但不阻断本次 PC 普通 IM 发布候选的服务端依赖：

- 会话置顶设置/取消接口。
- 会话免打扰持久化接口。
- 会话隐藏/删除/恢复接口及返回语义。
- 群公告读取接口或字段。
- 群成员群昵称、群主/管理员角色稳定字段。
- @ mentions 的发送、push、pull、history 回显字段确认。
- 回复引用历史摘要字段。
- 批量转发、批量删除接口及部分失败结构。

### 产品限制

- 历史搜索和类型筛选只覆盖当前已加载消息，不等同于全量服务端聊天记录搜索。
- 免打扰、隐藏/删除会话为本地 session 级能力，不承诺跨刷新、跨登录或跨设备保持。
- 会话置顶只读取服务端返回的 `isPinned`，PC 端暂不提供设置/取消置顶。
- 群公告没有接口时显示明确空状态。
- 批量转发、批量删除按单条接口顺序执行，不具备服务端批量原子性。

### 暂不处理项

- 不新增 App 能力。
- 不改在线客服临时会话。
- 不接入主题系统或换肤。
- 不扩展全量聊天历史服务端搜索。
- 不新增会话置顶/免打扰/隐藏恢复的假持久化逻辑。
- 不为服务端缺失字段造假展示。

## 发布候选判断

当前判断：可以进入 PC 普通 IM 发布候选冻结，前提是发布前回归命令和人工检查清单通过。

判断依据：

- 无已知 P0。
- 无阻断性 P1。
- 未读/已读专项测试覆盖并通过。
- 消息收发、normalize 和消息类型兜底专项测试覆盖并通过。
- 图片/文件发送、上传失败、预览、下载专项测试覆盖并通过。
- 收藏、撤回、删除、转发 refetch 一致性专项测试覆盖并通过。
- 批量转发/批量删除部分失败提示专项测试覆盖并通过。
- 服务端缺口已独立记录，且没有伪造前端持久化。

## 当前可发布范围

- PC 普通 IM 私聊和群聊。
- 会话列表：全部、未读、好友、群聊筛选；未读红点；最近消息；草稿提示；置顶排序读取。
- 聊天区：文本、emoji、图片、文件、名片、群事件、不支持类型兜底。
- 消息操作：回复、复制、转发、多选、撤回、删除、收藏、翻译、媒体下载。
- 图片/文件：按钮选择、拖拽、粘贴发送；上传失败提示；图片加载过程图和预览。
- 群聊基础：群 sender 可读展示、群事件系统提示、群成员列表、@ 人输入基础。
- 历史查看：当前已加载范围内的搜索、类型筛选、结果跳转。
- 未读/已读：微信式规则和“↑↑ N 条新消息”提示。

## 不包含范围

- App 端任何修改。
- 在线客服临时会话。
- 主题系统、换肤、视觉系统重构。
- 管理后台、客服分配、客服接待状态。
- 全量服务端历史搜索。
- 服务端缺失接口的假持久化能力。
- 群公告编辑、群管理高级能力。
- 批量转发/批量删除服务端原子事务。

## 已知限制

- 免打扰、隐藏/删除会话只做本地 session 级处理。
- 回复引用依赖消息数据中携带摘要；只有 `replyToMessageId` 时无法恢复引用内容。
- @ mentions 已发送 `mentions` 字段，但完整提醒规则依赖服务端和多端回显。
- 历史筛选只对当前加载消息生效。
- 批量操作有部分失败提示，但没有服务端统一事务。
- 群公告缺接口时只展示空状态。

## 必跑回归命令

只跑 PC 普通 IM 专项，不跑全量：

```bash
npm run typecheck
npx playwright test tests/browser/workspace-smoke.spec.ts -g "PC ordinary IM closure|PC ordinary IM stage two|PC ordinary IM stage three|PC ordinary IM stage four|updates IM unread|marks incoming messages" --workers=1
```

如发布候选期间修复了新的边界问题，需要补最小专项测试，并额外跑对应 `-g`。

## 发布前人工检查清单

### 未读/已读

- 未打开会话收到消息，只增加未读，不调用 markRead。
- 点击会话后红点清除，并显示“↑↑ N 条新消息”。
- 当前打开会话收到实时消息，直接展示且不产生未读红点。
- 自己发送的消息不产生未读。

### 私聊

- 文本、emoji、图片、文件、名片展示正常。
- 回复、复制、翻译、收藏、转发、撤回、删除可用。
- refetch 或切换会话后，收藏/撤回/删除/转发状态不回退。

### 群聊

- 群 sender 显示可读姓名，不显示 UUID。
- 群事件显示为系统提示，不进入普通用户气泡。
- 群成员列表和角色字段可读；缺字段时不造假。
- @ 人输入和发送回显正常。

### 图片/文件

- 图片和文件可选择、拖拽、粘贴发送。
- 上传失败有明确提示，待发送附件不丢。
- 图片加载过程图、预览、下载可用。
- 文件名、大小和下载可用。

### 历史和批量

- 历史搜索、类型筛选和结果跳转可用。
- 搜索无结果和加载失败状态清晰。
- 多选进入/取消/选择/取消选择可用。
- 批量转发、批量删除部分失败时有明确数量提示。

### 文档和支持

- 试用说明、反馈闭环、服务端缺口文档均为最新。
- 服务端缺口没有被前端假逻辑掩盖。
- 问题反馈模板包含会话 ID、消息 ID、requestId 和截图/日志。

## 回滚/暂停发布条件

满足任一条件，应暂停发布或回滚到上一稳定版本：

- 出现误已读或未读红点被错误清除。
- 非当前会话收到消息不显示或未读不增加。
- 当前打开会话实时消息不展示。
- 文本、图片、文件、名片、群事件出现空白消息。
- 图片/文件发送失败无提示或附件丢失。
- 撤回、删除、收藏、转发 refetch 后状态回退。
- 群聊 sender 大面积显示 UUID。
- 批量转发/批量删除静默失败。
- 401/403/500 错误无法通过 requestId 定位。
- 服务端返回结构变化导致 normalize 大面积失效。

## 发布候选冻结规则

- 冻结后只接受 P0/P1 修复、明确数据兼容修复、回归测试修复和文档勘误。
- 不新增功能。
- 不改 App、在线客服、主题系统。
- 不做大重构。
- 每个业务代码修复必须补最小专项测试，并跑必跑回归命令。
