# PC 端文档巡检机制

日期：2026-05-29

## 目标

让重构文档持续反映真实代码行为，避免任务矩阵、ADR、验证记录和 README 逐渐失真。

## 当前脚本

| 命令 | 用途 |
| --- | --- |
| `npm run docs:check` | 硬校验：必要文档、README 链接、任务状态、验证记录、ADR 结构。 |
| `npm run docs:garden` | 巡检报告：在硬校验基础上输出待开始、待处理、TODO/FIXME、历史参考等可能过期线索。 |

## 巡检频率

- 每完成一个阶段：必须运行 `npm run docs:garden`。
- 每次大批量重构后：必须检查任务矩阵和 README 当前阶段状态。
- 引入新架构决策后：必须补 ADR 或明确无需 ADR 的理由。

## 修复原则

1. 先修任务矩阵状态，再修 README 总览。
2. 已完成任务必须有验证记录。
3. ADR 只记录长期决策，不记录普通实现细节。
4. 巡检脚本只报告，不自动改文档；修改必须由 Codex 或工程师确认 diff。

## 后续升级

- 可在 CI 中增加 `npm run docs:garden`，初期只作为 report。
- 可增加更强的交叉链接校验，例如任务编号必须能反查验证记录和 ADR。
- 可增加“代码行为 vs 文档描述”的专项巡检脚本。
