# P27-IM-001 Query Scope Validation

日期：2026-06-07

范围：普通 IM 消息 query key、发送/cache mutation 写入、Gateway IM invalidation scope。

## 变更摘要

1. 活跃会话消息读取从 `pcQueryKeys.imMessages(apiBaseUrl, tenantToken, ...)` 改为 `pcQueryKeys.imMessagesForSession(session, ...)`。
2. `messageCacheMutationModel` 中本地失败、发送态、媒体态、转发插入等 cache 写入统一走 session/workspace scoped message key。
3. `invalidateMessages` 支持传入 session；有 session 时只失效当前 workspace 的 `pc-im-messages` 和 `pc-im-conversations`。
4. Gateway IM invalidation 支持 scopeKey；Gateway router 和 IM read delivery 传入当前 workspace scope，避免同 conversationId 跨 workspace 被一起失效。
5. 新增 `message-query-key-scope.spec.ts` 和 scoped invalidation 单测，防止运行时路径退回旧 token-scoped helper。
