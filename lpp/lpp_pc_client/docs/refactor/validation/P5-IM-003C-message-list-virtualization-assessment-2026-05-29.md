# P5-IM-003C Message List Virtualization Assessment

日期：2026-05-29

## 结论

当前不引入虚拟列表依赖，也不自研虚拟列表。

理由：

1. 当前阶段目标是页面瘦身和边界治理，不是性能专项。
2. 消息气泡包含图片、视频、本地上传状态、引用、事件消息、动态高度，虚拟化容易引入滚动定位和高度测量复杂度。
3. 未建立消息列表性能基线，无法判断引入虚拟化的收益与回归。
4. 用户明确要求不重复造轮子；如果后续确实需要虚拟化，应先做技术选型确认。

## 建议后续路径

在 `P8-PERF-003` 中执行：

1. 建立性能预算：例如 200、1000、5000 条消息的渲染耗时、滚动 FPS、内存占用。
2. 先优化低风险项：图片预加载、媒体缩略图、减少 render 派生、memo 化稳定 props。
3. 若仍不达标，再评估成熟库：
   - `@tanstack/react-virtual`
   - `react-virtuoso`
4. 引入前必须确认：
   - 动态高度支持。
   - 滚动到底行为。
   - 定位到 messageId。
   - 未读跳转。
   - 图片/视频加载后高度变化。
   - Windows Electron 兼容性。

本任务是评估任务，无代码改动要求。当前结论已记录，不新增依赖，不改变技术栈。
