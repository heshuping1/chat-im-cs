# PC 端复杂任务执行计划规范

日期：2026-05-29

## 目标

复杂重构不能只靠聊天上下文推进，必须有可查的计划、进度和决策记录。

## 什么时候需要计划

满足任一条件就需要计划：

1. 跨 3 个以上模块。
2. 涉及技术选型、新依赖、安全边界或核心链路删除。
3. 预计需要多个会话完成。
4. 需要阶段性验收或回滚策略。

## 计划类型

| 类型 | 使用场景 | 位置 |
| --- | --- | --- |
| 轻量计划 | 单会话内可完成，影响范围清楚 | 当前聊天 + 任务矩阵状态 |
| 版本化执行计划 | 跨会话、跨模块、风险较高 | `docs/superpowers/plans/` |

## 版本化计划最低内容

1. 背景和目标。
2. 范围和不做什么。
3. 分阶段任务清单。
4. 验收命令和手工验证。
5. 决策日志。
6. 风险与回滚。
7. 当前进度。

## 执行规则

- 每完成一个子任务，必须更新任务矩阵或计划进度。
- 每个 P0/P1 核心链路任务必须有验证记录。
- 发现计划和代码现实冲突时，先更新计划，再继续实现。
- 新会话默认先读 `AGENTS.md`、README、任务矩阵和当前计划。
