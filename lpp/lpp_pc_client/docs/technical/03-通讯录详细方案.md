# 通讯录详细方案

状态：当前有效

日期：2026-06-05

适用范围：`lpp_pc_client/src/renderer/contacts`

## 1. 范围

通讯录覆盖联系人、好友、群、客户、员工、组织条目、联系人详情、好友申请提醒和邀请二维码入口。

## 2. 主要代码入口

| 能力 | 入口 |
| --- | --- |
| 页面入口 | `src/renderer/components/ContactsPage.tsx` |
| 联系人目录 | `src/renderer/data/contact-directory.ts` |
| 权限规则 | `src/renderer/data/contact-directory-permissions.ts` |
| 添加好友 | `src/renderer/contacts/hooks/useContactAddFriendController.ts` |
| 目录控制 | `src/renderer/contacts/hooks/useContactsDirectoryController.ts` |
| 好友申请提醒 | `src/renderer/contacts/hooks/useFriendRequestReminderController.ts` |
| 联系人详情 | `src/renderer/components/ContactDetailViews.tsx` |
| 侧边面板 | `src/renderer/components/ContactSidePanel.tsx` |
| 邀请二维码 | `src/renderer/components/ContactsInviteQrCard.tsx` |

## 3. 数据流

1. 通讯录按当前身份、空间和权限加载联系人数据。
2. 不同联系人类型统一为 `ContactItem` 展示。
3. 详情页按联系人类型展示不同字段。
4. 好友申请提醒进入侧边栏空间提醒汇总。
5. 邀请二维码从账号入口或通讯录入口触发真实接口。

## 4. 核心约束

1. 不同联系人类型不能混淆业务含义。
2. 客户资料和普通联系人资料字段不能互相伪造。
3. 好友申请状态以服务端为准。
4. 组织、员工、客户字段缺失时展示空态或已有字段，不造假。
