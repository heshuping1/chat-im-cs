# 设置与诊断详细方案

状态：当前有效

日期：2026-06-05

适用范围：`lpp_pc_client/src/renderer/settings`

## 1. 范围

设置与诊断覆盖 PC 端外观、通知、网络线路、诊断记录、诊断包导出、聊天背景、归档、桌面通知、任务栏提醒、强制更新和 Electron 桌面能力边界。

## 2. 主要代码入口

| 能力 | 入口 |
| --- | --- |
| 设置模型 | `src/renderer/data/settings/pc-settings.ts` |
| 设置 store | `src/renderer/data/settings/settings-store.ts` |
| 设置页组件 | `src/renderer/settings/components/` |
| 网络线路 | `src/renderer/data/network/site-line-manager.ts` |
| 诊断记录 | `src/renderer/data/diagnostics/` |
| 诊断导出 | `src/renderer/settings/runtime/diagnosticsExport.ts` |
| 桌面通知 | `src/renderer/data/reminder/` |
| 强制更新 | `src/renderer/settings/runtime/clientUpdateRuntime.ts`、`src/renderer/App.tsx` |
| 桌面 API 合同 | `src/shared/desktop-api.ts` |
| Electron 主进程 | `src/main/` |

## 3. 数据流

1. 设置项由 settings store 读取和持久化。
2. App 根节点根据主题、字体、密度、边界和动效设置写入 dataset。
3. 通知、提醒和任务栏状态由 reminder service 根据当前模块、窗口焦点和用户设置决定。
4. API、Gateway、消息、客服、运行错误等诊断进入 diagnostics 模块。
5. 诊断导出通过桌面 API 生成本地文件或诊断包。
6. 强制更新通过更新状态订阅显示遮罩，并执行下载、安装或退出。

## 4. 核心约束

1. 新增桌面能力必须通过 preload/desktop API 合同，不直接在 renderer 暴露 Electron IPC。
2. 诊断导出必须考虑脱敏，不能泄露 token、密码或真实敏感数据。
3. 通知策略必须避免当前打开会话重复提醒。
4. 强制更新遮罩不能被普通交互绕过。
5. 网络线路切换必须清理 query cache 并同步 auth session。
