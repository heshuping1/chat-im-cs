# 在线客服详细方案

状态：当前有效

日期：2026-06-05

适用范围：`lpp_pc_client/src/renderer/customer-service`

## 1. 范围

在线客服覆盖客服、管理员和所有者可见的 PC 业务工作台能力，包括会话列表、排队/活跃会话、接待状态、消息区、发送状态、消息动作、快捷回复、知识库抽屉、AI 建议入口和提醒。

不覆盖完整管理后台、复杂客户运营后台和服务端未提供的数据闭环。

## 3. 数据流

1. 侧边栏按角色判断是否允许读取客服工作台。
2. 工作台拉取 active、queue、summary 等线程数据。
3. 打开线程后拉取线程详情和消息。
4. 客服发送消息进入发送控制器和 outbox 恢复流程。
5. 接待状态通过 reception status 接口读写，并影响自动/手动接队模式。
6. Gateway 或轮询更新未读、排队、SLA 风险和桌面提醒。

## 4. 核心约束

1. 没有客服权限的角色不能看到在线客服工作台。
2. 接待状态和接队模式以服务端确认为准，前端乐观更新失败必须回滚。
3. 客服消息动作必须走权限模型，不在组件里绕过权限判断。
4. 客服线程 unread 和 active 状态不能和普通 IM 混用。
5. 快捷回复、知识库和 AI 建议依赖服务端数据，不造假内容。

## 5. 服务端依赖

在线客服依赖：

1. `customerServiceThreads`
2. `customerServiceReceptionStatus`
3. `customerServiceThreadDetail`
4. `customerServiceThreadMessages`
5. `customerServiceThreadAction`
6. `quickReplies`
7. `knowledgeSearch`
8. `aiSuggestion` / `aiSuggestions`

如新增客服动作、状态、字段或线程类型，必须同步更新 API 合同、normalizer、功能矩阵和服务端支持文档。
