## 6. 新需求闭环

新增需求按这个顺序处理：

1. 在 `01-PC端需求规格说明.md` 登记需求。
2. 在 `requirements/01-PC端需求池.md` 补齐详细需求。
3. 补齐需求状态和设计状态；只有 `需求状态：完成` 且 `设计状态：完成`，才允许进入开发。
4. 在 `03-PC端功能矩阵.md` 新增或更新功能行。
5. 如涉及接口或字段，更新 `05-服务端支持.md`。
6. 如涉及模块边界、数据流或技术策略，更新 `04-PC端技术方案总览.md` 或 `technical/` 对应详细方案。
7. 实现后补最小测试和验证命令。
8. 发布前更新 `06-PC端发布文档.md`，发布候选前复核 `release/01-PC普通IM发布候选检查.md`。

开发准入硬规则：

1. `需求状态` 可选值：`待补齐` / `评审中` / `完成` / `变更中` / `暂缓`。
2. `设计状态` 可选值：`待补齐` / `评审中` / `完成` / `变更中` / `暂缓`。
3. 只有 `需求状态：完成` 且 `设计状态：完成` 可以进入开发。
4. 需求或设计进入 `变更中` 后，相关功能必须暂停新增实现，只允许做明确的 P0/P1 修复或回滚。
5. 如果确认不需要新增技术方案，也必须把 `设计状态` 标为 `完成`，并说明“沿用现有方案”。
