# PC IM + 在线客服场景验收矩阵

日期：2026-06-02

## 1. 使用方式

本矩阵用于证明闭环方案是否真的完整。每个场景都必须能回答：

- 输入是什么。
- 期望行为是什么。
- 归属是谁。
- 未读怎么变化。
- 是否提醒。
- 哪个模型负责解释。
- 需要哪些测试覆盖。

状态说明：

- `已覆盖`：已有明确代码模型和单测。
- `需补测试`：代码方向存在，但缺专门测试。
- `需补实现`：当前方案要求存在，但代码还没有完整落地。
- `依赖服务端`：需要服务端协议或接口补齐。

## 13. 完整性结论

当前文档闭环后，场景覆盖结论如下：

- IM 基础收发、当前/非当前会话未读、read receipt、自己消息：方向完整，已有较多测试。
- 在线客服基础访客/客服自己消息、菜单不清未读、线程详情清未读：方向完整，已有核心测试。
- Push/delivery/gap guard：方向完整，已有核心测试。
- SnapshotReconcileService：已在 M05 落地并覆盖旧 snapshot 不覆盖 push、弱字段补齐、客服 ledger 合并等核心场景。
- 服务端真实 cursor/afterSeq gap sync：当前 API 不支持，依赖服务端，是成熟系统最终必要条件。
