模块编号：M13

模块名称：整体验收

模块职责：
- 对 M00-M12 的模块化重构做全量测试、类型检查和构建验收。
- 汇总非服务端依赖项和服务端缺口，避免把当前 API 不支持能力伪造在 PC 端。
- 给出本轮开发分支是否可提交/继续手动验收的明确结论。

验收输入：
- `tests/unit` 全量单元测试。
- `npm run typecheck`。
- `npm run build`。
- `03-场景验收矩阵.md` 与服务端缺口文档。

验收输出：
- 通过/失败命令与关键结果。
- 未提交文件范围。
- 服务端缺口说明。

边界：
- 不在 M13 引入新业务实现。
- 发现失败时只修复与 M00-M12 相关的问题。
- 手动验收项只能列为待人工确认，不能伪造通过。

测试：
- `npx.cmd vitest run tests/unit`
- `npm.cmd run typecheck -- --pretty false`
- `npm.cmd run build`

验收记录（2026-06-03）：
- `npx.cmd vitest run tests/unit` 通过，153 个测试文件 / 833 个测试。
- `npm.cmd run typecheck -- --pretty false` 通过。
- `npm.cmd run build` 通过；`@microsoft/signalr` PURE 注释告警为依赖构建告警，不影响产物。
- `03-场景验收矩阵.md` 已按 M00-M12 结果校准：已覆盖项、服务端缺口、延期专项分开标记。
- 手动验收仍待人工确认，不能在 M13 自动化验收中伪造通过。

未完成声明原因：
- IM / 在线客服真实多端、多账号、长连接恢复、附件发送等场景仍需在可用服务端环境做手动验收。
- 客服高级状态、撤回/编辑事件、精确 cursor/afterSeq gap sync 属于服务端事件契约或后续专项，不在 PC 端伪造。
