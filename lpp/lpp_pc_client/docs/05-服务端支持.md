# PC 普通 IM 服务端支持

状态：当前有效入口

日期：2026-06-06

适用范围：`lpp_pc_client`

## 1. 定位

本文档是服务端依赖和缺口看板。它只保留接口、字段、状态和关联文档；详细行为设计见对应 `technical/` 文档。

PC 端不会用前端假持久化掩盖服务端缺口。

## 2. 已有真实接口

| 能力 | Endpoint |
| --- | --- |
| 私聊消息列表 | `GET /api/client/v1/direct-chats/{conversationId}/messages` |
| 群聊消息列表 | `GET /api/client/v1/groups/{conversationId}/messages` |
| 私聊发送消息 | `POST /api/client/v1/direct-chats/{conversationId}/messages` |
| 群聊发送消息 | `POST /api/client/v1/groups/{conversationId}/messages` |
| 上传媒体 | `POST /api/client/v1/media/upload` |
| 私聊已读 | `POST /api/client/v1/direct-chats/{conversationId}/read` |
| 群聊已读 | `POST /api/client/v1/groups/{conversationId}/read` |
| 撤回 | `POST /api/client/v1/messages/{messageId}/recall` |
| 删除消息 | `POST /api/client/v1/messages/{messageId}/delete` |
| 转发 | `POST /api/client/v1/messages/forward` |
| 收藏 | `POST /api/client/v1/favorites`、`GET /api/client/v1/favorites/list` |
| 翻译 | `POST /api/client/v1/translate/message`、`POST /api/client/v1/translate/text` |
| 群资料相关 | `GET /api/client/v1/groups/{conversationId}`、`GET /api/client/v1/groups/{conversationId}/members`、`GET /api/client/v1/groups/{conversationId}/announcements` |

## 4. P1 接口最小合同

| 能力 | Request | Response |
| --- | --- | --- |
| 置顶 | `{ "isPinned": true }` | `{ "conversationId": "...", "isPinned": true, "updatedAt": "..." }` |
| 免打扰 | `{ "isMuted": true }` | `{ "conversationId": "...", "isMuted": true, "updatedAt": "..." }` |
| 隐藏/恢复 | `{ "hidden": true }` | `{ "conversationId": "...", "hidden": true, "updatedAt": "..." }` |
| 本机已落库搜索 | PC 本地库 query: `keyword/messageType/cursor/limit/fromTime/toTime` | 本机 SQLite FTS 结果；不要求服务端全文搜索 |
| 批量转发 | `{ "messageIds": ["m1"], "targetConversationId": "c2" }` | `{ "successIds": ["m1"], "failedItems": [] }` |
| 批量删除 | `{ "messageIds": ["m1", "m2"] }` | `{ "successIds": ["m1"], "failedItems": [{ "messageId": "m2", "message": "..." }] }` |
| @ mentions | `{ "mentions": [{ "userId": "...", "displayName": "..." }] }` | send/push/pull/history 保持同结构 |
| 回复引用 | `{ "replyToMessageId": "m1" }` | `{ "replyToMessageId": "m1", "reply": { "messageId": "m1", "senderDisplayName": "...", "preview": "...", "messageType": "text" } }` |

## 5. 更新规则

1. 新增服务端依赖时，先在本文档增加一行。
2. 如果对应需求未完成，不得标记 `开发准入：准入`。
3. 接口字段变化时，同步更新 `src/renderer/data/api/endpoints.ts`、API client、normalizer 和测试。
4. 缺口关闭后，把 `闭环状态` 从 `待排期` 更新为 `已完成`，并回填 `03-PC端功能矩阵.md`。
5. 跨设备全量历史搜索不在当前规划内，不新增服务端全文搜索缺口。
