### PC-REQ-20260606-001：Gateway CORS / Origin 与真实 push 联调闭环

- 优先级：P0
- 类型：接口变更 / 持续开发
- 当前行为：PC 已有 Gateway 连接、事件路由、IM side effects 和 gap sync 补偿；Gateway 失败不阻塞页面。
- 期望行为：真实环境允许 PC 完成 `/ws/client/negotiate` 和 `/ws/client` 连接，真实接收 `msg.new`、`msg.read` 并更新消息、未读、已读和提醒。
- 交付范围：服务端 CORS/Origin 放行；真实 push 联调；PC 沿用现有 Gateway 架构。
- 不交付范围：不改 cookie 鉴权；不新增第二套实时通道；不以 mock push 替代真实联调。
- 依赖接口：`/ws/client/negotiate`、`/ws/client`。
- 涉及模块：`src/renderer/components/GatewayBridge.tsx`、`src/renderer/data/gateway/`、`src/renderer/data/im-read/`、`src/renderer/messages/`。
- 需求状态：完成
- 设计状态：完成
- 关联设计文档：`../technical/05-普通IM-P0实时与未读一致性详细方案.md`、`../05-服务端支持.md`
- 开发准入：准入
- 验收标准：Gateway connected；`msg.new` 实时展示；非当前会话增加未读；当前会话不产生未读；`msg.read` 更新 read receipt；断线重连触发 gap sync。
- 必测命令：`npm run typecheck`、`npm run test:core`、普通 IM unread/push Playwright smoke。
- 风险：服务端 CORS/Origin、Authorization、SignalR negotiate 配置不一致会导致真实联调失败。
- 实现状态：已完成

### PC-REQ-20260606-002：自己消息未读一致性服务端修正

- 优先级：P0
- 类型：接口变更 / 持续开发
- 当前行为：PC 已有 read model、本地 readSeq 补偿、发送成功后 markRead、Gateway self message 识别和展示兜底。
- 期望行为：服务端在 conversation、push、pull/history 中稳定返回 self/sender/direction 字段；`lastMessage` 是当前账号自己发送时返回 `unreadCount: 0`。
- 交付范围：服务端修正 self/sender 字段和 unread；PC 复用现有 read model、Gateway side effects、send succeeded read 补偿。
- 不交付范围：不在前端伪造服务端持久化 readSeq；不承诺未完成协议闭环的全平台强一致。
- 依赖接口：`GET /api/client/v1/conversations`、普通 IM push、普通 IM pull/history、普通 IM read 接口。
- 涉及模块：`src/renderer/data/im-read-model.ts`、`src/renderer/data/im-read/`、`src/renderer/data/gateway/gateway-im-side-effects.ts`、`src/renderer/messages/models/messageConversationListModel.ts`。
- 需求状态：完成
- 设计状态：完成
- 关联设计文档：`../technical/05-普通IM-P0实时与未读一致性详细方案.md`、`../05-服务端支持.md`
- 开发准入：准入
- 验收标准：自己发送不产生未读；同账号另一设备发送不产生未读；self lastMessage refetch 后未读不复活；他人非当前会话新消息仍产生未读。
- 必测命令：`npm run typecheck`、`npm run test:core`、普通 IM unread Playwright smoke。
- 风险：服务端字段不稳定会导致 PC 只能本地兜底，无法证明多端一致性。
