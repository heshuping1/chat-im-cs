## 5. 后续设计准入要求

上述需求均已完成需求登记，但仍处于 `设计状态：待补齐`，因此不得直接进入代码实现。详细设计必须补齐以下内容后才能准入：

1. 每项能力的 PC 端信息架构：坐席工作台、管理员/所有者视角、数据中心、Widget 端各自承接什么。
2. DDD 边界：conversation、message、transfer、typing preview、read receipt、recall、reporting、configuration 分别由哪个 owner 承载。
3. API 与 Gateway 事件归一方案：不得在页面组件中拼 endpoint 或解释 raw WS payload。
4. 状态归属：服务端快照、UI 状态、运行时瞬时状态、本地补偿状态必须分开。
5. 验证方案：contract/normalizer、model、Gateway side effects、权限、人工联调场景逐项列出。
