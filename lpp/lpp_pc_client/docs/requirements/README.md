# PC 端需求目录

状态：当前有效入口

日期：2026-06-06

适用范围：`lpp_pc_client`

## 1. 定位

本目录保存详细需求正文。根目录 `01-PC端需求规格说明.md` 只保留需求索引和准入规则。

## 2. 文档

| 文档 | 用途 |
| --- | --- |
| `01-PC端需求池.md` | P0/P1 需求详情、验收标准、依赖接口和实现状态 |
| `02-PC群功能任务清单.md` | 群功能 API 支持、前端状态、联调状态和下一步任务 |
| `03-PC普通IM会话与批量接口任务清单.md` | 会话持久化和 batch 接口的 API 支持、前端状态、联调状态和下一步任务 |

## 3. 维护规则

1. 新需求先登记到 `01-PC端需求池.md`。
2. 需求状态和设计状态都为 `完成` 后，才能在根目录 `01-PC端需求规格说明.md` 标记 `开发准入：准入`。
3. 需求影响功能状态时，同步更新 `03-PC端功能矩阵.md`。
4. 需求影响服务端接口时，同步更新 `05-服务端支持.md`。
5. 需求影响模块边界或技术策略时，同步更新 `04-PC端技术方案总览.md` 或 `technical/` 对应详细方案。
